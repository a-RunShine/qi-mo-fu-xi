# db-question-suite

数据库学科选择题套件。给定章节(第X章),**支持两种工作模式**——**生成**题目(.md)或**写解析**(.md),由 Pre-flight 自动判断。

## Pre-flight check (必做,失败立即报错)

1. **确认学科**:解析用户提到的章节名 → 必须是数据库学科(章号一/四/五/七/八/十/十一)。若用户提到"软件工程""Java""面向对象" → **不使用本 skill**。
2. **确认工作模式**:
   - 用户说"出题 / 选择题 / 生成 / 改造" → **GEN 模式**(生成题目)
   - 用户说"写解析 / 补解析 / 改解析" → **ANALYSIS 模式**(写解析)
   - 同时说两者 → 先 GEN 再 ANALYSIS(顺序)
3. **GEN 模式前检查**:
   - 规范路径:`ls docs/guides/选择题深度规格说明.md` 存在
   - 产物目录:`ls 数据库/复习题/选择题/` 存在
4. **ANALYSIS 模式前检查**:
   - 题目文件存在:`ls 数据库/复习题/选择题/第X章_XXX_选择题.md` 存在;若不存在 → 报错"请先 GEN 模式"
   - 规范路径:`ls docs/guides/选择题解析深度规格说明.md` 存在
   - 解析文件未存在 / 问用户:覆盖 / 跳过 / `_v2`

## When to use

Use for:

- "出数据库第四章 10 道选择题"(GEN)
- "生成数据库第七章数据库设计的选择题"(GEN)
- "写数据库第四章选择题解析"(ANALYSIS)
- "给数据库第七章选择题补解析"(ANALYSIS)
- "改造数据库第五章选择题(深度题比例太低)"(GEN 重做)
- "改造解析(基础题改深度题)"(ANALYSIS 重做)

Do not use for:

- 练习题/综合应用题(直接按 `docs/guides/练习题深度规格说明.md`)
- 软件工程章节(用 `se-question-suite`)
- 知识点讲解/思维导图(用 `ai-lecture`)
- PDF 渲染(用 `md-pdf`)

## Output directory and file naming

按 CLAUDE.md 命名规范:

- GEN:`数据库/复习题/选择题/第X章_{章节名}_选择题.md`(+ 可选 .pdf)
- ANALYSIS:`数据库/复习题/选择题/第X章_{章节名}_选择题解析.md`(+ 可选 .pdf)

例:`数据库/复习题/选择题/第四章_数据库安全性_选择题.md` + `第四章_数据库安全性_选择题解析.md`

**已存在时三选一**:覆盖 / 跳过 / 新版本(`_v2`)。**禁止自动覆盖,必须问用户。**

## GEN 模式:生成选择题

### G.1 加载规范

Read `references/选择题深度规格说明.md`(软链接 → `docs/guides/选择题深度规格说明.md`)。

理解:

- 两类题(基础/深度)定义
- 5 种构造手法(操作序列/代码判断/相邻辨析/多约束权衡/复合分类)
- 4 条硬标准(字数差 ≤20%、措辞对称、结构同构、真实干扰)
- 4 步生成流程
- 7 大秒选反模式
- 学科适配(数据库哪些章节适合场景题)
- 格式规范(选择题文件)

### G.2 收集章节考点

- Read 现有 AI 讲解:`数据库/ai讲解/第X章_{章节名}_AI讲解.md`(如有)
- Read 章节 PDF 解析文本(如有,`数据库/突击笔记/pdf_images/`)
- 列出 18-28 个考点候选,标 ★★★(必考)与一般考点

### G.3 规划基础/深度配比

- 题目总数:10-18 道(默认 10,与现有样本对齐)
- 基础题 20-25%(纯记忆必考点,如范式定义、ACID 性质)
- 深度题 75-80%(操作序列/代码判断/相邻辨析/多约束权衡/复合分类)

### G.4 按 5 种构造手法生成深度题

每题按"4 步生成流程"构造:

```
1. 写题干场景(按 5 种构造手法选一种)
2. 列出 3-4 个"可能的错误理解路径"(干扰项素材源)
3. 把"正确机制"和"错误路径"用同构措辞写成 4 个选项
4. 自检:长度差、措辞对称、能否秒选
```

### G.5 写题目文件

- 标题:`# 第X章 {章节名} — 选择题`
- 顶部说明:`> 对应 数据库/复习题/选择题/第X章_XXX_选择题.md,共 N 道(基础 N₁ + 深度 N₂)。`
- 题号格式:`1. 题干内容（  ）` / `16.（多选）题干（  ）` / `3. ★★★ 题干（  ）`
- 末尾:`## 答案` + 行式 `1.X 2.X ...`

### G.6 GEN Quality checklist(每题自检)

- [ ] 题干有具体场景/代码/数据(非纯概念)?
- [ ] 正确项与干扰项措辞接近、差别在机制?
- [ ] 至少一个干扰项对应常见误解/相邻概念?
- [ ] 不可仅靠背诵答出?
- [ ] 满足 4 条中至少 2 条?
- [ ] 字数差 ≤ 20%?
- [ ] 7 大秒选反模式全避开?
- [ ] 格式:题号 `X.` 开头?题干末尾 `（  ）`?多选 `（多选）` 标在题号后?★★★ 后空格?

### G.7 可选:PDF 渲染

问用户"需要生成 PDF 吗?",用 `md-pdf` skill 转 PDF。

## ANALYSIS 模式:写选择题解析

### A.1 加载规范 + 读题目

- Read `references/选择题解析深度规格说明.md`(软链接 → `docs/guides/选择题解析深度规格说明.md`)
- Read 配套的 `数据库/复习题/选择题/第X章_XXX_选择题.md`
- 识别每题是**基础题**(纯记忆)还是**深度题**(满足 4 条中≥2 条)

### A.2 选项质量前置审查(解析深度的隐性前提)

**这是关键步骤,不能跳过**。解析能写多深取决于选项设计得多精。

对每道深度题,自问三问(强制):

1. 各选项字数差是否 ≤ 30%?(硬线 ≤20%)
2. 正确项能否被"更绝对 / 更长 / 更具体"秒选?
3. 至少一个干扰项是否对应真实误解(不是凑数)?

任一答案为"否" → **先切到 GEN 模式改题,再回 ANALYSIS 写解析**。

### A.3 按两类题两种形式生成解析

**基础题解析(精简式)**——省略正文,只保留答案 + 金句:

```
### 第 X 题

**答案：X**

> [金句:口诀/边界提醒]

---
```

**深度题解析(完整四段式)**——必含:

```
### 第 X 题

**答案：X**

**解析：**

[正文 8-15 行]

> [金句:口诀/类比/边界提醒]

---
```

### A.4 正文 5 条强制标准(深度题)

1. **首句点破考点**——"这题考的是 XXX",不先念题干
2. **理清逻辑链**——多步操作/多层概念用路径/列表/对比可视化
3. **逐选项审判**——正确项说"为什么对",错误项说"错在哪个具体点";**禁止只说"不选 X"**
4. **点穿陷阱**——至少一处揭示干扰项对应何种误解/相邻概念
5. **瑕疵题诚实标注**——双正确答案/教材口径分歧明说并解释取舍

### A.5 收尾金句(每题都要有)

三种形式任选或组合:

- **口诀**——把机制压缩成顺口溜
- **生活化类比**——审计=摄像头、角色=会员卡、视图=面具、加密=防线、CASCADE=砍树枝
- **边界提醒**——点出易混边界、相邻概念的分界线

### A.6 写解析文件

- 标题:`# 第X章 {章节名} — 选择题解析`
- 顶部说明:`> 对应 XXX_选择题.md,每题附详细解析与坑点说明。`
- 题号格式:`### 第 X 题`("第""X""题"间有空格)
- 文档结尾:`## 答案汇总`(数据库规范)+ 行式 `1.X 2.X ...` 或表格

### A.7 ANALYSIS Quality checklist(每题自检)

#### 选项质量(前置)

- [ ] 各选项字数差 ≤ 30%?(硬线 ≤20%)
- [ ] 正确项不能被"更绝对 / 更长 / 更具体"秒选
- [ ] 至少一个干扰项对应真实误解

#### 深度题正文

- [ ] 首句点破考点(不是先念题干)?
- [ ] 多步操作/多层概念用路径/列表/对比可视化?
- [ ] 正确项说"为什么对"(点出关键机制)?
- [ ] 每个错误项说"错在哪个具体点"(不是只说"不选")?
- [ ] 至少点穿一处陷阱(揭示对应何种误解/相邻概念)?
- [ ] 瑕疵题诚实标注并解释取舍?
- [ ] 有 `>` 金句(口诀/类比/边界提醒)?
- [ ] 篇幅 8-15 行,口语化 + 粗体关键词?
- [ ] PDF 兼容(无 `⭐⊂✓✗` 等非常用 Unicode)?

#### 基础题正文

- [ ] 省略正文,只保留答案 + 金句?
- [ ] 金句点出口诀/边界提醒?
- [ ] 篇幅 1-3 行?

#### 全篇

- [ ] 文档结尾有答案汇总(行式或表格)?
- [ ] 题号格式、标题格式统一?

### A.8 可选:PDF 渲染

用 `md-pdf` skill 转 PDF。

## Verification(两模式通用)

生成后必须执行:

1. Read 输出 .md,逐题对照对应模式的 Quality checklist
2. 用 `python3 ~/.claude/skills/md-pdf/scripts/check_residue.py {file}.md` 检查脚本残留
3. 启动 1 个 `explore` subagent 独立审稿(过审纪律:任何"过审"声明必须能回答谁审的 + 按什么标准 + 报告在哪)
4. 整理 subagent 报告中的"关键修改点清单"给用户,用户拍板后改题

## 过审纪律

选择题深度规格要求"切不可通过选项区别就选出答案"。自评容易漏,必须:

- 启动 1-2 个 `explore` subagent 独立审稿
- subagent 重点检查:干扰项是否对应真实误解、正确项能否被秒选、子集陷阱/补强陷阱、术语是否真实存在
- 拿到报告后整理"关键修改点清单"(每题 ≤ 3 条核心问题 + 改法)给用户过目
- 用户拍板后改题 → 重转 PDF

详细复盘见 `docs/retrospectives/选择题生成-选项质量自封过审-错误教训.md`。
