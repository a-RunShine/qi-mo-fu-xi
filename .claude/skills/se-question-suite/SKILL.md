# se-question-suite

软件工程学科选择题套件。给定章节(第X章),**支持两种工作模式**——**生成**题目(.md)或**写解析**(.md),由 Pre-flight 自动判断。

## Pre-flight check (必做,失败立即报错)

1. **确认学科**:解析用户提到的章节名 → 必须是软件工程学科(章号一/二/三/四/五/六)。若用户提到"数据库""Java""面向对象" → **不使用本 skill**。
2. **确认工作模式**:
   - 用户说"出题 / 选择题 / 生成 / 改造 / 出软工v2" → **GEN 模式**
   - 用户说"写解析 / 补解析 / 改解析 / v2解析" → **ANALYSIS 模式**
   - 同时说两者 → 先 GEN 再 ANALYSIS(顺序)
3. **确认版本**(v2 深度版 vs 原版):
   - 用户说"v2 / 深度版" → 走 `v2深度版/` 子目录
   - 用户说"原版" → 走父目录
   - 用户未说 → 默认走 v2 深度版(2026-06-25 后的新格式)
4. **GEN 模式前检查**:
   - 规范路径:`ls docs/guides/选择题深度规格说明.md` 存在
   - 产物目录:`ls 软件工程/复习题/选择题/(或 v2深度版/)` 存在
5. **ANALYSIS 模式前检查**:
   - 题目文件存在:`ls 软件工程/复习题/选择题/第X章_XXX_选择题.md`(或 v2)存在;若不存在 → 报错"请先 GEN 模式"
   - 规范路径:`ls docs/guides/选择题解析深度规格说明.md` 存在
   - 解析文件未存在 / 问用户:覆盖 / 跳过 / `_v2`

## When to use

Use for:

- "出软工第四章 10 道选择题"(GEN)
- "生成软件工程第二章可行性研究的题"(GEN)
- "出软工第三章 v2 深度题"(GEN,v2 深度版)
- "写软工第四章选择题解析"(ANALYSIS)
- "给软工第二章 v2 深度题补解析"(ANALYSIS)
- "改造软工解析(基础题改深度题)"(ANALYSIS 重做)

Do not use for:

- 数据库章节(用 `db-question-suite`)
- 知识点讲解/思维导图(用 `ai-lecture`)
- PDF 渲染(用 `md-pdf`)

## Output directory and file naming

按 CLAUDE.md 命名规范:

- GEN(原版):`软件工程/复习题/选择题/第X章_{章节名}_选择题.md`
- GEN(v2):`软件工程/复习题/选择题/v2深度版/第X章_{章节名}_选择题_v2.md`
- ANALYSIS(原版):`软件工程/复习题/选择题/第X章_{章节名}_选择题解析.md`
- ANALYSIS(v2):`软件工程/复习题/选择题/v2深度版/第X章_{章节名}_选择题解析_v2.md`
- 配套 PDF(可选):同目录 `.pdf`

**v2 深度版**:2026-06-25 后的深度题升级版(每章 10 题,深度题占比更高)。

**已存在时三选一**:覆盖 / 跳过 / 新版本(`_v3` 等)。**禁止自动覆盖,必须问用户。**

## GEN 模式:生成选择题

### G.1 加载规范

Read `references/选择题深度规格说明.md`(软链接 → `docs/guides/选择题深度规格说明.md`)。

### G.2 收集章节考点

- Read 现有 AI 讲解:`软件工程/ai讲解/第X章_XXX_AI讲解.md`(如有)
- Read 现有 v2 深度版:`软件工程/复习题/选择题/v2深度版/第X章_XXX_选择题_v2.md`(若存在,作为深度题样本)
- 列出 18-28 个考点候选,标 ★★★(必考)与一般考点

### G.3 规划基础/深度配比

- 题目总数:10-22 道(v2 深度版统一 10 题,原版 10-22 道)
- 基础题 20-25%(内聚耦合类型排序、CMM 级数、α/β 测试地点等)
- 深度题 75-80%(操作序列/代码判断/相邻辨析/多约束权衡/复合分类)

### G.4 按 5 种构造手法生成深度题

软工 5 种构造手法的典型应用:

- **操作序列题**——可行性研究多方案权衡、维护工单分类
- **代码/伪代码判断题**——内聚耦合类型判断、McCabe 覆盖级别
- **相邻概念辨析题**——条件覆盖 vs 判定覆盖、顺序内聚 vs 通信内聚
- **多约束冲突权衡题**——过程模型选用(风险/工期/需求稳定性)
- **复合场景分类题**——风险策略判主次、维护类型分类

每题按"4 步生成流程"构造(写题干场景 → 列错误路径 → 同构措辞写选项 → 自检)。

### G.5 写题目文件

- 标题:`# 第X章 {章节名} — 选择题`
- 顶部说明:`> 对应 软件工程/复习题/选择题/第X章_XXX_选择题.md,共 N 道(基础 N₁ + 深度 N₂)。`
- 题号格式:`1. 题干内容（  ）` / `16.（多选）题干（  ）` / `3. ★★★ 题干（  ）`
- 末尾:`## 答案` + 行式 `1.X 2.X ...`

### G.6 GEN Quality checklist(每题自检)

- [ ] 题干有具体场景/代码/数据(非纯概念)?
- [ ] 正确项与干扰项措辞接近、差别在机制?
- [ ] 至少一个干扰项对应常见误解/相邻概念?(软工尤其注意内聚耦合类型的相邻辨析)
- [ ] 不可仅靠背诵答出?
- [ ] 满足 4 条中至少 2 条?
- [ ] 字数差 ≤ 20%?
- [ ] 7 大秒选反模式全避开?
- [ ] 格式:题号 `X.` 开头?题干末尾 `（  ）`?多选 `（多选）` 标在题号后?★★★ 后空格?

### G.7 可选:PDF 渲染

问用户"需要生成 PDF 吗?",用 `md-pdf` skill 转 PDF。

## ANALYSIS 模式:写选择题解析

### A.1 加载规范 + 读题目

- Read `references/选择题解析深度规格说明.md`(软链接 → `docs/guides/选择题解析深度规格说明.md`)
- Read 配套的题目文件(原版或 v2)
- 识别每题是**基础题**(纯记忆)还是**深度题**(满足 4 条中≥2 条)

### A.2 选项质量前置审查(解析深度的隐性前提)

对每道深度题,自问三问(强制):

1. 各选项字数差是否 ≤ 30%?(硬线 ≤20%)
2. 正确项能否被"更绝对 / 更长 / 更具体"秒选?
3. 至少一个干扰项是否对应真实误解(不是凑数)?

任一答案为"否" → **先切到 GEN 模式改题,再回 ANALYSIS 写解析**。

### A.3 按两类题两种形式生成解析

**基础题解析(精简式)**——省略正文,只保留答案 + 金句:

```
### 第 X 题

**答案：X**

> [金句:口诀/边界提醒]

---
```

**深度题解析(完整四段式)**——必含:

```
### 第 X 题

**答案：X**

**解析：**

[正文 8-15 行]

> [金句:口诀/类比/边界提醒]

---
```

### A.4 正文 5 条强制标准(深度题)

1. **首句点破考点**——"这题考的是 XXX",不先念题干
2. **理清逻辑链**——多步操作/多层概念用路径/列表/对比可视化
3. **逐选项审判**——正确项说"为什么对",错误项说"错在哪个具体点";**禁止只说"不选 X"**
4. **点穿陷阱**——至少一处揭示干扰项对应何种误解/相邻概念
5. **瑕疵题诚实标注**——双正确答案/教材口径分歧明说并解释取舍

### A.5 软工专属收尾金句素材

软工解析常用类比(收尾金句素材库):

- 顺序内聚 = 流水线(工人处理零件)
- 通信内聚 = 工人处理同一批零件
- 控制耦合 = 指挥棒(flag 决定执行路径)
- 信息隐藏 = 模块只暴露"做什么",藏起"怎么做"和内部数据
- 扇出/扇入 = 管孩子/被爹叫
- 传话游戏 = 需求分析
- 体检项目 = 软件测试
- 保险 = 风险策略

**口诀示例**——内聚 7 级从低到高:"偶逻时过通顺功",最后一个"功"=功能内聚最高。

### A.6 写解析文件

- 标题:`# 第X章 {章节名} — 选择题解析`
- 顶部说明:`> 对应 XXX_选择题.md,每题附详细解析与坑点说明。`
- 题号格式:`### 第 X 题`("第""X""题"间有空格)
- 文档结尾:`## 答案速查`(软工规范)+ 行式 `1.X 2.X ...` 或表格

### A.7 ANALYSIS Quality checklist(每题自检)

#### 选项质量(前置)

- [ ] 各选项字数差 ≤ 30%?(硬线 ≤20%)
- [ ] 正确项不能被"更绝对 / 更长 / 更具体"秒选
- [ ] 至少一个干扰项对应真实误解

#### 深度题正文

- [ ] 首句点破考点(不是先念题干)?
- [ ] 多步操作/多层概念用路径/列表/对比可视化?
- [ ] 正确项说"为什么对"(点出关键机制)?
- [ ] 每个错误项说"错在哪个具体点"(不是只说"不选")?
- [ ] 至少点穿一处陷阱(揭示对应何种误解/相邻概念)?
- [ ] 瑕疵题诚实标注并解释取舍?
- [ ] 有 `>` 金句(口诀/类比/边界提醒)?软工优先用概念类比(流水线/指挥棒/传话游戏)
- [ ] 篇幅 8-15 行,口语化 + 粗体关键词?
- [ ] PDF 兼容(无 `⭐⊂✓✗` 等非常用 Unicode)?

#### 基础题正文

- [ ] 省略正文,只保留答案 + 金句?
- [ ] 金句点出口诀/边界提醒?
- [ ] 篇幅 1-3 行?

#### 全篇

- [ ] 文档结尾有答案速查(行式或表格)?
- [ ] 题号格式、标题格式统一?

### A.8 可选:PDF 渲染

用 `md-pdf` skill 转 PDF。

## Verification(两模式通用)

生成后必须执行:

1. Read 输出 .md,逐题对照对应模式的 Quality checklist
2. 用 `python3 ~/.claude/skills/md-pdf/scripts/check_residue.py {file}.md` 检查脚本残留
3. 启动 1 个 `explore` subagent 独立审稿
4. 整理 subagent 报告中的"关键修改点清单"给用户

## 过审纪律

软工第三章 v2 2026-06-25 曾自封"过审",但 7 个设计问题全漏(子集陷阱/不存在术语/缺瑕疵标注/唯一特殊项/补强陷阱/篇幅超限/列表行可合并),必须 subagent 独立审稿。

- 启动 1-2 个 `explore` subagent 独立审稿
- subagent 重点检查:干扰项是否对应真实误解、正确项能否被秒选、子集陷阱/补强陷阱、术语是否真实存在
- 拿到报告后整理"关键修改点清单"(每题 ≤ 3 条核心问题 + 改法)给用户过目
- 用户拍板后改题 → 重转 PDF

详细复盘见 `docs/retrospectives/选择题生成-选项质量自封过审-错误教训.md`。
