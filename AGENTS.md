## 学科目录（结构因学科而异，勿混用）

## 技能（.opencode/skills/）

### md-pdf — MD 转 PDF（Claude 深棕 + 信纸色主题）
主能力：MD 渲染为带项目主题色的 PDF。辅以 PDF OCR（Swift+Vision）、题目合并/残留检查、横线格信纸模板生成。
工作流：MD 渲染 →（可选 OCR/合并/残留检查/信纸模板）。命令从工作区根目录执行：

- OCR：`swift .opencode/skills/md-pdf/scripts/ocr_pdf.swift input.pdf output_dir output_ocr.txt`
- 合并：`python3 .opencode/skills/md-pdf/scripts/merge_qa.py base.md supplement.md final.md [--title "标题"]`
- 残留检查：`python3 .opencode/skills/md-pdf/scripts/check_residue.py final.md`
- MD 转 PDF：`python3 .opencode/skills/md-pdf/scripts/convert_md_to_pdf.py input.md [output.pdf]`
- 横线格信纸：`python3 .opencode/skills/md-pdf/scripts/make_lined_paper.py -o L|P [-n 行数]`

### ai-lecture — 知识点转教师风格讲解 + 思维导图
工作流：确定来源 → 分析知识结构 → 生成讲解 MD → 可选转 PDF → 可选生成思维导图。

- 思维导图渲染：`dot -Tpng -Gdpi=200 input.dot -o output.png`（可选 SVG：`dot -Tsvg input.dot -o output.svg`）
- DOT 模板：`.opencode/skills/ai-lecture/templates/mindmap_template.dot`
- 输出目录按学科区分：
  - 数据库 → `数据库/ai讲解/`（AI讲解 平铺根目录）+ `数据库/ai讲解/思维导图/`（.dot/.png）
  - 软件工程 → `软件工程/ai讲解/`（.md）+ `软件工程/思维导图/`（.dot/.png）

## 依赖
- `pip install 'fpdf2>=2.5.0'`（MD 转 PDF，低于 2.5.0 缺 `collection_font_number` 会失败）
- `pip install 'Pillow>=10.0'`（处理装饰图标：白底透明化 + 缩放到 256×256）
- Graphviz `dot` >= 7.0（思维导图用 `splines=ortho`，低版本不支持）
- macOS 中文字体（脚本硬编码路径）：
  - `/System/Library/Fonts/STHeiti Medium.ttc`
  - `/System/Library/Fonts/Supplemental/Songti.ttc`
- 非 macOS：需修改 `convert_md_to_pdf.py` 字体路径
- PDF 不支持 `⭐⊂✓✗` 等 Unicode，用 `【重点】` / `是...的子集` / `正确` / `错误` 替代

## PDF 主题色（2026-06-22 起生效）
所有 PDF 默认采用「Claude 深棕 + 信纸色」主题，由
`.opencode/skills/md-pdf/scripts/convert_md_to_pdf.py` 控制：

| 元素 | 色值 | 用途 |
|---|---|---|
| 背景 | `#FFFBF5` | 全页底色（信纸色） |
| 主标题 H1 | `#D97707` | 强调色（琥珀橙） |
| H2/H3 | `#5C3A21` | 副标题（中棕） |
| H4 | `#785537` | 小标题（浅棕） |
| 正文 | `#3D2B1F` | 深棕黑 |
| 页脚 | `#C2680A` | 章节名 + 页码 |
| 表格表头底 | `#F0E4C9` | 浅米 |
| 代码块底 | `#F5EAD0` | 浅米 |
| 边框/分隔线 | `#D9C4A2` | 卡其 |

- **装饰图标**：`assets/ornament.png`（页脚左侧，赤陶色放射花纹，PIL 抠白底处理；非 Claude 商标）。缺失时页脚自动降级为纯文字。
- **章节名推断**：从 Markdown 文件名 `第X章_XXX_类型.md` 提取，自动写入页脚左侧。无匹配时第一页省略页脚。
- **题号间空行**：`_preprocess_lines` 检测"上一题选项 → 下一题题干"的空行，用 `ln(5mm)` 加大间距；普通空行 `ln(2mm)`。
- **页脚 `header()`/`footer()`**：每页自动绘全页背景 + 页脚（装饰图标 + 章节名 + 页码）；`bottom_margin=24` 给页脚留位。

## PDF 维护
- **备份目录**：`pdf_backup_YYYYMMDD/`（按日期滚动的 PDF 旧主题色快照）。当前无活跃备份，需要回滚时执行 `cp -r pdf_backup_YYYYMMDD/* .`
- **批量重生成**：从工作区根执行

      find ./数据库 ./软件工程 ./面向对象原则 -name "*.md" -not -name "INDEX.md" \
        | xargs -I{} python3 .opencode/skills/md-pdf/scripts/convert_md_to_pdf.py {}

- **未纳入脚本的 PDF**：`数据库/突击笔记/` 下 4 个 PDF（`大题解题方法` / `SQL语言` / `选填题知识点` / `范式级别判断`）是 OCR 源素材，无 .md 源，未走新主题色。需要重新 OCR + 整理为 .md 后才能用脚本生成。
- **章节名匹配失败**：_extract_chapter_name 依赖 `第X章_XXX_` 命名规范，不符合命名的 .md 生成 PDF 时页脚会留空（不影响内容）。

## 选择题深度规格
完整规范见 `docs/guides/选择题深度规格说明.md`，生成选择题时必须遵守。核心要点：

- **基础题占 20-25%**：只保留纯记忆必考点（★★★及核心概念辨析）
- **深度题占 75-80%**：满足以下 4 条中至少 2 条：
  1. 场景化题干（操作序列/代码/数据/角色/图形，非纯概念）
  2. 选项微差（正确项与干扰项差别在机制细节）
  3. 合理陷阱（干扰项对应常见误解或相邻概念）
  4. 不可仅靠背诵（需推理或应用）
- **5 种构造手法**：操作序列题、代码判断题、相邻概念辨析题、多约束权衡题、复合场景分类题
- **参考样本**：`数据库/复习题/选择题/第四章_数据库安全性_选择题.md`
- 每章基础题数量参考：18题章留4-5题；16题章留3-4题；28题章留6题左右

## 选择题解析深度规格
完整规范见 `docs/guides/选择题解析深度规格说明.md`，生成选择题解析时必须遵守。核心要点：

- **两类题两种形式**：基础题（纯记忆型）省略正文，只保留 `**答案：X**` + `> 金句`；深度题完整四段式（`### 第X题` → `**答案：X**` → `**解析：**` → 正文 → `> 金句` → `---`）
- **正文 5 条强制标准（深度题）**：
  1. 首句点破考点（"这题考的是 XXX"，不先念题干；基础题由金句承担点题）
  2. 理清逻辑链（含多步操作/多层概念时，用路径/列表/对比可视化；纯辨析题可省）
  3. 逐选项审判（正确项说"为什么对"，错误项说"错在哪个具体点"，禁只说"不选"；强干扰项可标注"（强干扰项）"）
  4. 点穿陷阱（至少一处揭示干扰项对应何种误解/相邻概念，而非仅说错在哪）
  5. 瑕疵题诚实标注（双正确答案/教材口径分歧等明说并解释取舍）
- **收尾金句**：每题都要有；口诀 / 生活化类比 / 边界提醒（审计=摄像头、角色=会员卡、视图=面具、加密=最后防线、CASCADE=砍树枝）；可选延伸或跨题关联
- **答案汇总**：`## 答案汇总`（数据库）或 `## 答案速查`（软工），行式 `1.X 2.X …` 或表格（题量多时）
- **风格篇幅**：中文口语化 + 粗体关键词；"为什么"驱动不堆定义；深度题 8-15 行，基础题金句 1-3 行；PDF 兼容（无 `⭐⊂✓✗`）
- **参考样本**：`数据库/复习题/选择题/第四章_数据库安全性_选择题解析.md` + `软件工程/复习题/选择题/第四章_软件设计_选择题解析.md`
- 适用范围：数据库 + 软件工程各章 `*_选择题解析.md`

## 选择题格式规格
选择题与解析文件的排版格式，对齐数据库参考样本。生成/改造时必须遵守。

### 选择题文件格式
- **题号直接在题干前**：`1. 题干内容（  ）`（数字编号 + 点 + 空格 + 题干）
- **答题括号**：题干末尾加 `（  ）`（全角括号 + 两个空格），跨行题干括号放在问句末尾而非引导句后
- **多选标注**：题号后加 `（多选）`，如 `16.（多选）下列...（  ）`；单选不加标记
- **★★★标记**：放在题号后、题干前，加空格，如 `3. ★★★ 下列内聚...（  ）`；不写"必考/坑点/计算"等文字
- **无标题、无分隔线**：不写 `### 第X题` 标题，题间不加 `---` 分隔线
- **文件结构**：`# 标题` → `> 说明` → 题目列表 → `## 答案` → 答案行（`1.X 2.X ...`）

### 解析文件格式
- **每题标题**：`### 第 X 题`（"第"后空格、数字两边空格、"题"前空格）
- **答案**：`**答案：X**`（中文冒号）
- **解析**：`**解析：**`
- **点睛金句**：`> ` 引用块
- **题间分隔**：`---` 分隔线
- **答案速查**：文末 `## 答案速查` + 表格（`| 题号 | 1 | 2 | ... |`、`| 答案 | X | Y | ... |`）

### 跨行题干括号位置（易错）
题干跨多行时（含代码块、任务表、场景描述），括号放在**问句末尾**，不放引导句后：
- 正例：`阅读以下伪代码：` + 代码块 + `下列说法正确的是：（  ）`
- 反例：`阅读以下伪代码：（  ）` + 代码块 + `下列说法正确的是：`
