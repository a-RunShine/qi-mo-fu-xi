**主题参数**（`--theme`）：

## 选择题深度规格

完整规范见 `docs/guides/选择题深度规格说明.md`，生成选择题时必须遵守。核心要点：- **基础题占 20-25%**：只保留纯记忆必考点（★★★及核心概念辨析）
- **深度题占 75-80%**：满足以下 4 条中至少 2 条：
  1. 场景化题干（操作序列/代码/数据/角色/图形，非纯概念）
  2. 选项微差（正确项与干扰项差别在机制细节）
  3. 合理陷阱（干扰项对应常见误解或相邻概念）
  4. 不可仅靠背诵（需推理或应用）
- **5 种构造手法**：操作序列题、代码判断题、相邻概念辨析题、多约束权衡题、复合场景分类题
- **选项设计硬标准**（核心原则：切不可通过选项区别就选出答案！）：
  - **七大秒选反模式（必须杜绝）**：长度失衡（正确项更长）/ 绝对化偏置（正确项用"必须"干扰项用"可能"）/ 结构不对称（一项完整句子其他术语）/ 包含性元标记（"等""包括"）/ 否定独苗 / 唯一特殊项（数字/代码/外文）/ 子集陷阱（A 是 B 的子集）
  - **4 条硬标准**：字数差 ≤ 20% / 限定词与动词结构对齐 / 全术语或全描述同构 / 每个干扰项对应真实误解
  - **4 步生成流程**：写题干场景 → 列出 3-4 个"可能的错误理解路径" → 与正确项用同构措辞写成 4 个选项 → 自检秒选风险
- **参考样本**：`数据库/复习题/选择题/第四章_数据库安全性_选择题.md`
- 每章基础题数量参考：18题章留4-5题；16题章留3-4题；28题章留6题左右


## 选择题解析深度规格

完整规范见 `docs/guides/选择题解析深度规格说明.md`，生成选择题解析时必须遵守。核心要点：

- **选项质量前置审查（解析的隐性前提）**：解析能写多深取决于选项设计得多精。**先过选项质量关，再写深度解析**——读者只靠字面特征秒选了，解析写得再透也没用。三问不过不写：

  1. 各选项字数差是否 ≤ 30%？
  2. 正确项能否被"更绝对 / 更长 / 更具体"秒选？（必须不能）
  3. 至少一个干扰项是否对应真实误解（不是凑数）？

  - 不过关 → 改题（首选）/ 降基础题 / 标"瑕疵题"诚实标注
- **两类题两种形式**：基础题（纯记忆型）省略正文，只保留 `**答案：X**` + `> 金句`；深度题完整四段式（`### 第X题` → `**答案：X**` → `**解析：**` → 正文 → `> 金句` → `---`）
- **正文 5 条强制标准（深度题）**：

  1. 首句点破考点（"这题考的是 XXX"，不先念题干；基础题由金句承担点题）
  2. 理清逻辑链（含多步操作/多层概念时，用路径/列表/对比可视化；纯辨析题可省）
  3. 逐选项审判（正确项说"为什么对"，错误项说"错在哪个具体点"，禁只说"不选"；强干扰项可标注"（强干扰项）"）
  4. 点穿陷阱（至少一处揭示干扰项对应何种误解/相邻概念，而非仅说错在哪）
  5. 瑕疵题诚实标注（双正确答案/教材口径分歧等明说并解释取舍）
- **收尾金句**：每题都要有；口诀 / 生活化类比 / 边界提醒（审计=摄像头、角色=会员卡、视图=面具、加密=最后防线、CASCADE=砍树枝）；可选延伸或跨题关联
- **答案汇总**：`## 答案汇总`（数据库）或 `## 答案速查`（软工），行式 `1.X 2.X …` 或表格（题量多时）
- **风格篇幅**：中文口语化 + 粗体关键词；"为什么"驱动不堆定义；深度题 8-15 行，基础题金句 1-3 行；PDF 兼容（无 `⭐⊂✓✗`）
- **参考样本**：`数据库/复习题/选择题/第四章_数据库安全性_选择题解析.md` + `软件工程/复习题/选择题/第四章_软件设计_选择题解析.md`
- 适用范围：数据库 + 软件工程各章 `*_选择题解析.md`

## 选择题过审纪律（自评 ≠ 审稿）

选择题/解析题生成后，**禁止**自封"过审"。机械指标用脚本，设计质量必须由 **subagent 独立审稿**。

### 强制过审流程

1. **生成完选择题/解析题后**，必须用 `task` 工具启动 1-2 个 `explore` subagent 独立审稿（按 `docs/retrospectives/选择题生成-选项质量自封过审-错误教训.md` 里的 prompt 模板）
2. **拿到审稿报告后**，整理"关键修改点清单"（每题 ≤ 3 条核心问题 + 改法）给用户过目
3. **用户拍板后**改题 → 重转 PDF
4. 任何回复里写"已过审"必须能回答：**谁审的**（subagent? 用户?）+ **按什么标准审**（规范文档第 X 节）+ **报告在哪**（可链接/可粘贴）

### 脚本能查 vs 脚本不能查


| 脚本能查（机械指标）                     | 不能查（设计质量，必须 subagent）                                                                                     |
| ---------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| 字数差、格式统一、多选标注位置、题号连号 | 干扰项是否对应真实误解、正确项能否被秒选、子集陷阱 / 补强陷阱、术语是否真实存在、解析首句是否点破、逐选项审判是否到位 |

### 教训案例

2026-06-25 软件工程第三章 v2 生成时自封"选项质量过审"——脚本只查了字数，但 **7 个设计问题**全漏（子集陷阱 / 不存在术语 / 缺瑕疵标注 / 唯一特殊项 / 补强陷阱 / 篇幅超限 / 列表行可合并），必须 subagent 独立审才能发现。详细复盘见 `docs/retrospectives/选择题生成-选项质量自封过审-错误教训.md`。

## 产出物审查机制（自评 ≠ 审稿）

选择题过审纪律只覆盖选择题/解析题，本章节扩展为**所有产物**的通用审查机制。

### 原则

- **自评 ≠ 审稿**：认知偏差不可避免，自评时"看自己孩子"容易觉得都好，必须由独立主体审查
- **脚本只查机械指标**：字数差、格式统一、题号连号等脚本可查；设计质量（干扰项是否对应真实误解、答案是否正确、解析是否到位）必须 subagent
- **"过审"必须有外部依据**：回复中写"已过审"必须能回答——谁审的、按什么标准、报告在哪

### 适用范围

- 选择题/解析题
- 练习题（含综合应用题）
- AI 讲解（ai-lecture 产物）
- 思维导图（DOT+PNG）
- 任何带"我审过了"字样的输出

### 强制过审流程（5 步）

1. **生成后** → 启动 1-2 个 explore subagent 独立审稿
2. **拿到报告** → 整理"关键修改点清单"（每题 ≤ 3 条核心问题 + 改法）给用户过目
3. **用户拍板** → 改题 → 重转 PDF
4. **必要时多轮审查**：一次审查可能不够，复杂产物（如综合应用题）可能需要 3-5 轮迭代
5. **"过审"声明**：必须能回答——谁审的（subagent? 用户?）、按什么标准（规范文档第 X 节）、报告在哪（可链接/可粘贴）

### 常见致命问题清单（7 类）

| # | 问题类型 | 典型表现 | 预防方法 |
|---|----------|----------|----------|
| 1 | 答案错误 | 综合题验算漏了、陷阱子问推理错 | 闭包 + Chase 定理双验证 |
| 2 | 答案-解析矛盾 | 修改答案时忘同步解析 | 改答案后必须 grep 解析中所有引用 |
| 3 | 复制粘贴残留 | "沿用上次"、重复金句 | 生成后全文搜索"沿用""上次"等关键词 |
| 4 | 字数差 > 20% | 最长选项 vs 最短选项比例失衡 | 脚本统计字数差 |
| 5 | 选项结构不对称 | 分组效应（A/B 一组 vs C/D 一组） | 检查 4 个选项是否同构 |
| 6 | 基础题比例失衡 | 100% 全基础 vs 应 20-25% | 生成前规划基础/深度配比 |
| 7 | Unicode 警告字符 | ⭐✓✗❌⋈⁺ⱼ 等 | 转 PDF 前 grep 特殊符号 |

### 教训案例（大题解题方法 5 轮审稿）

| 轮次 | 得分 | 关键发现 | 修复 |
|------|------|----------|------|
| 一审 | 37/50 | 第 4 题答案-解析矛盾 | 改答案 B→A |
| 二审 | 39/50 | 基础题比例 57% 超标 | 第 1/5 题场景化改造 |
| 三审 | 42/50 | 范式方向写反、字数失衡 | 1NF⊂2NF→BCNF⊂3NF |
| 四审 | 35/50 | 第(5)子问答案错误（陷阱子问） | 修正表格法推导 |
| 五审 | 45.5/50 | 达标投用 | — |

**核心教训**：一次审查不够。陷阱子问/复杂推理题答案必须**双重验证**（闭包 + Chase 定理双验证）。新增子问尤其危险——新增内容往往比原有内容更容易出错。

## 选择题格式规格

选择题与解析文件的排版格式，对齐数据库参考样本。生成/改造时必须遵守。

### 选择题文件格式

- **题号直接在题干前**：`1. 题干内容（  ）`（数字编号 + 点 + 空格 + 题干）
- **答题括号**：题干末尾加 `（  ）`（全角括号 + 两个空格），跨行题干括号放在问句末尾而非引导句后
- **多选标注**：题号后加 `（多选）`，如 `16.（多选）下列...（  ）`；单选不加标记
- **★★★标记**：放在题号后、题干前，加空格，如 `3. ★★★ 下列内聚...（  ）`；不写"必考/坑点/计算"等文字
- **无标题、无分隔线**：不写 `### 第X题` 标题，题间不加 `---` 分隔线
- **文件结构**：`# 标题` → `> 说明` → 题目列表 → `## 答案` → 答案行（`1.X 2.X ...`）

### 解析文件格式

- **每题标题**：`### 第 X 题`（"第"后空格、数字两边空格、"题"前空格）
- **答案**：`**答案：X**`（中文冒号）
- **解析**：`**解析：**`
- **点睛金句**：`> ` 引用块
- **题间分隔**：`---` 分隔线
- **答案速查**：文末 `## 答案速查` + 表格（`| 题号 | 1 | 2 | ... |`、`| 答案 | X | Y | ... |`）

### 跨行题干括号位置（易错）

题干跨多行时（含代码块、任务表、场景描述），括号放在**问句末尾**，不放引导句后：

- 正例：`阅读以下伪代码：` + 代码块 + `下列说法正确的是：（  ）`
- 反例：`阅读以下伪代码：（  ）` + 代码块 + `下列说法正确的是：`

### 参考样本

- 题目：`数据库/复习题/选择题/第四章_数据库安全性_选择题.md`
- 解析：`数据库/复习题/选择题/第四章_数据库安全性_选择题解析.md`
