# 练习题深度规格说明

本规范用于约束 AI 生成练习题(综合应用题 + 大题专项)时的质量标准。所有练习题生成任务必须遵守。

适用范围:数据库 `数据库/复习题/练习题/` 与 `数据库/复习题/大题专项/` 下的 `*_练习题.md` 产物。

参考样本:

- 综合应用题:`数据库/复习题/练习题/大题解题方法_练习题.md` + `大题解题方法_练习题解析.md`
- 板块分组:`数据库/复习题/练习题/关系代数SQL综合_练习题.md`(10 大题,基础+提高)
- 专项题:`数据库/复习题/大题专项/大题专项_练习题.md`(11 大题,5 板块)

---

## 一、两类题的定义

### 综合应用题(模拟卷形式)

模拟真实考试题型组合,**选择题 + 大题混合**,按考试时间命名(例:"58 分钟版""35 分钟版")。

特征:

- 顶部声明分值与时间(如 `7 题 × 2 分 = 14 分` / `58 分钟`)
- 选择题部分用 `## 一、` 编号;大题部分用 `## 二、`
- 末尾有「答案速查」

**典型样本**:大题解题方法(7 选 + 5 大题)、范式级别判断(8 选 + 5 填空 + 1 大题)

### 大题专项(按板块分组)

按主题板块分组,**全部大题无选择题**,按难度递增或主题聚类。

特征:

- 按 `## 板块一/二/三/...` 划分
- 每个板块 2-5 大题
- 题目文件顶部声明总题数与总分(如 `11 大题 / 140 分`)
- 关系模式定义在板块顶部集中给出

**典型样本**:大题专项(11 大题/5 板块)、关系代数SQL综合(10 大题/2 板块:基础+提高)

## 二、5 种构造手法

### 1. 范式判断题(L/R/LR 划分 + 闭包 + Chase 定理)

**结构**:给定关系模式 U 和函数依赖集 F,要求:

- (1) L/R/LR 类属性划分(3 分)
- (2) 求所有候选码(3 分)
- (3) 求属性组闭包(4 分)
- (4) 判断范式级别(1NF/2NF/3NF/BCNF)
- (5) BCNF 分解(10 分)

**陷阱**:范式递进方向写反(1NF⊂2NF⊂3NF⊂BCNF,范围递减)、L 类属性遗漏、闭包计算漏步。

**双验证**:Chase 定理验证无损连接 + 闭包验证依赖保持(任一缺失即扣分)。

### 2. 关系代数求表达式

**结构**:给定关系模式(原题要写明 S/C/SC/T/TC 等),用 π/σ/⋈/÷/G/F 符号完成查询。

**关键标志词对应**(必背):

- "全部" → 除运算(÷)
- "否定/没有" → 差运算(-)
- "至少/全部满足" → 除运算变体
- "最多/不超过" → NOT EXISTS 子查询

**陷阱**:投影与选择的顺序写反、自然连接漏条件、聚合前未分组。

### 3. SQL 编写题

**结构**:把关系代数翻译为 SQL,或直接用嵌套/聚合/连接写查询。

**关键模式**:

- `IN` / `NOT IN` / `EXISTS` / `NOT EXISTS` 嵌套
- `GROUP BY` + `HAVING` 聚合
- `JOIN` 多表连接(注意内外连接选择)
- 子查询 vs 连接的性能权衡(题目不考性能,选可读性高的)

**陷阱**:`COUNT(*)` 与 `COUNT(列)` 区别(空值)、`WHERE` 与 `HAVING` 误用、相关子查询漏条件。

### 4. ER 图设计题

**结构**:给定业务场景(银行/电商/学校/医院),设计 ER 图,标注实体/属性/联系/基数。

**陷阱**:1:1 / 1:n / m:n 联系画错、属性错放(应该是属性画成实体)、弱实体漏双线框。

**配套**:大题专项_ER图设计.md 应有 .dot 文件(Graphviz)或 .drawio 源文件。

### 5. 事务并发题(2PL/三级封锁/死锁/活锁)

**结构**:给事务操作序列(读/写/加锁/释放),判断:

- 是否遵守 2PL(所有加锁在所有释放之前)
- 三级封锁协议哪一级
- 是否产生死锁/活锁
- 可串行化判断(冲突等价于某个串行序列)

**陷阱**:2PL 与三级封锁协议的方向写反(三级是 2PL 的加强版/特例,而非反之)、死锁与活锁混淆(死锁=循环等待/活锁=无限等同一资源)。

## 三、答案自检清单(双验证)

练习题大题答案**必须双重验证**,单次推导可能漏陷阱子问:

### 验证 A:闭包验证

- 候选码穷举:遍历所有最小属性组合,验证闭包是否 = U
- 闭包逐步推导:每一步写出用的依赖
- 范式判断:每级范式独立判定(不能跳级)

### 验证 B:Chase 定理验证

- 无损连接分解:画初始表格,逐条依赖追"a"行,有全 a 行即无损
- 依赖保持:每个原依赖能否从分解后依赖推出
- 任一不通过 → 分解方案不完美,需调整

### 陷阱子问专项检查

- (5)(6)(7) 等高位子问(陷阱多)
- "修正方案""基于(1)结果" 等依赖子问(易忘前文)
- "说明理由"类(不能只给结论,必须写过程)

### 自检流程

```
生成大题答案
  ├─ 闭包/Chase 双验证
  ├─ 陷阱子问专项检查
  ├─ 与第(1)(2)(3)子问结论一致性核查
  └─ 教材口径分歧时标"以张海藩/王珊版为准"
```

**教训**:大题解题方法 5 轮审稿案例(见 `docs/retrospectives/` 软工第 5 轮:第(5)子问答案错误,由闭包 + Chase 漏验导致)。

## 四、格式规范

### 题目文件

- 标题:`# {主题} —— 练习题{（XX 分钟版）}`
- 顶部说明:`> 本卷共 N 大题,按...排序,总计 X 分。**不计分、不限时**,供集中突破使用。`
- 板块分组:`## 板块一/二/...` + `### 第 N 题(主题,分值)`
- 子问:`（1）... （3 分）` / `（2）... （5 分）`
- 关系模式定义:在板块顶部用 `>` 引用块集中给出
- 末尾:`## 答案速查` + 表格或行式汇总

### 解析文件

- 标题:`# {主题} —— 练习题解析`
- 区块:`### 第 N 题` → `**答案：X**` → `**解析：**` → 正文 → `> 金句` → `---`
- 答案+解析+金句 三件套(无简化版,基础题也不省)
- 子问标注:`（1）` / `（2）` 分点回答

### 关系模式符号

- 主键用 `**属性名**`(粗体),下划线在 PDF 中自动处理
- 实体用 `学生 S(学号, 姓名, 性别, ...)` 形式
- 联系用 `选课 SC(学号, 课程号, 成绩)` 形式
- 函数依赖用 `F={A→B, BC→D, ...}` 集合形式

## 五、参考样本与适用范围

| 题型 | 样本文件 | 特点 |
|---|---|---|
| 模拟卷 | `大题解题方法_练习题.md` + 解析 | 选+大题混合,分值时间标注 |
| 范式专项 | `范式级别判断_练习题.md` | 选+填+大题,集中突破 |
| 关系代数+SQL | `关系代数SQL综合_练习题.md` | 原创场景,基础+提高 2 板块 |
| 板块分组大题 | `大题专项/大题专项_练习题.md` | 5 板块 11 大题,含 ER 图 |
| 章节配套练习 | `第X章_XXX_练习题.md` | 36-44 题,选择题为主 |

**适用范围**:以上 5 类均用本规范,新生成练习题默认按"大题专项"格式(板块分组,纯大题)。

## 六、生成时的检查清单

生成每份练习题后,逐条自检:

- [ ] 顶部声明总题数 / 总分 / 不计分不限时?
- [ ] 板块分组清晰(每板块 2-5 大题)?
- [ ] 关系模式定义在板块顶部集中给出?
- [ ] 子问分值标注完整(每问注明分数)?
- [ ] 大题答案**双验证**(闭包 + Chase 定理)?
- [ ] 陷阱子问专项检查(高位子问、依赖子问)?
- [ ] 解析含答案+解析+金句三件套?
- [ ] 教材口径分歧时标"以张海藩/王珊版为准"?
- [ ] 末尾有答案速查(表格或行式)?
- [ ] PDF 兼容(无 `⭐⊂✓✗` 等非常用 Unicode)?
