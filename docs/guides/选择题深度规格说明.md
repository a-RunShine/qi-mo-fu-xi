# 选择题深度规格说明（题库生成规范）

本规范用于约束 AI 生成选择题时的质量标准。所有选择题生成任务必须遵守。

参考样本：`数据库/复习题/选择题/第四章_数据库安全性_选择题.md`

---

## 一、两类题的定义

### 基础题（记忆型）

纯靠背诵即可作答的必考点。特征：

- 题干是概念直接提问（"下列X中，最高的Y是"）
- 选项是并列的事实列举，无机制关联
- 正确项与干扰项边界清晰，一眼可辨
- 不涉及应用、推理、判断

**典型**：内聚最高=功能内聚；McCabe公式 V(G)=E-N+2；CMM共5级；α测试地点=开发环境。

### 深度题（理解应用型）

需要理解机制、结合场景推理才能作答。满足以下 4 条中**至少 2 条**：

1. **场景化题干**：有具体操作序列 / 代码 / 数据 / 角色 / 图形，而非纯概念
2. **选项微差**：正确项与干扰项措辞接近，差别在机制细节
3. **合理陷阱**：干扰项错得有道理，对应常见误解或相邻概念
4. **不可仅靠背诵**：套口诀/定义无法直接得出，需推理或应用

---

## 二、核心区别对照

| 维度 | 基础题 | 深度题 |
|---|---|---|
| 题干 | "下列X属于Y的是" | "某系统执行了①②③④操作后，结果是" |
| 信息量 | 一句话 | 多步操作/代码片段/数据表/图形 |
| 选项关系 | 并列事实 | 同一机制的几种可能表现，需辨析 |
| 干扰项设计 | 明显错误 | 对应常见误解或相邻概念 |
| 作答路径 | 回忆→匹配 | 读场景→套机制→排除→选 |
| 改一个字能否区分 | 否 | 是（陷阱就在一个词的差别） |

---

## 三、比例规范

- **基础题占 20-25%**：只保留纯记忆必考点（标★★★及核心概念辨析）
- **深度题占 75-80%**：其余全部深化
- 每章基础题数量参考：18题章留4-5题；16题章留3-4题；28题章留6题左右

## 四、深度题的 5 种构造手法

### 1. 操作序列题（机制追踪）

给一串操作，问最终状态。考机制传播路径。

**数据库样本第1题**：DBA授权→U1转授U2→DBA又授U2→REVOKE CASCADE，问U2最终有无权限。陷阱在"CASCADE只回收U1传播的，DBA直接授予的保留"。

### 2. 代码/伪代码判断题

给真实代码片段，判断属于哪类。

**软工第四章**：给"A调用B传flag，B内部分支"的伪代码，判耦合类型。陷阱设"flag也是数据，为何不是数据耦合"——因为控制耦合的flag决定执行路径，更危险。

### 3. 相邻概念辨析题（最见功力）

给易混的两个概念的混合特征，让区分。

**软工第五章第10题**：条件覆盖 vs 判定覆盖"不可比"，反直觉，需用 if(a>0&&b>0) 反例推理。

**软工第四章**：给"顺序处理+共享数据"的模块，判顺序内聚还是通信内聚（两者相邻易混）。

### 4. 多约束冲突权衡题

给多个互相冲突的项目约束，让在多个方案间权衡。

**软工第一章**：军工高风险（→螺旋）+ 需求较明确（→瀑布）+ 6个月出第一版（→增量），让权衡。干扰项设"螺旋因风险分析最合适"（忽略工期）、"增量大但风险分析不足"（忽略风险）。

### 5. 复合场景分类题

给一个含多项的清单/工单，逐项分类，设跨类陷阱。

**软工第五章**：给5条维护工单逐条分类，陷阱"交付前发现的bug算改正性维护吗"（不算，交付前是开发阶段）。

**软工第六章**：给"买保险+增加测试+接受损失"复合风险策略，判主策略和次策略。

## 五、学科适配（哪些考点适合深度题）

### 天然适合场景题（必须深化）

| 章节 | 考点 | 深化方向 |
|---|---|---|
| 第一章 | 过程模型选用 | 多约束冲突权衡 |
| 第二章 | DFD平衡、经济可行性、成本估算 | 给图判平衡、给数据算NPV、给多语言比FP |
| 第三章 | 需求分类、用例关系、需求验证 | 给混合需求条目、给矛盾需求判维度 |
| 第四章 | 内聚耦合判断、McCabe覆盖 | 给伪代码判类型、给流程图算V(G)并判覆盖级别 |
| 第五章 | 逻辑覆盖、等价类边界值、维护类型 | 给代码判覆盖级别、给复杂输入算等价类、给工单清单分类 |
| 第六章 | 关键路径、PERT、风险策略、CMM | 给AOE网络图算、给复合策略判、给组织描述判级别 |

### 只能做基础题（保留记忆）

- 内聚耦合的类型排序（最高/最低）
- McCabe公式中E、N的含义
- 三种基本控制结构
- CMM级数
- α/β测试地点与主体
- 数据字典/DFD符号
- 软件危机表现（纯列举）

## 六、正例与反例

### 正例（数据库第四章第1题）

某公司DBA执行：①GRANT SELECT ON Employee TO U1 WITH GRANT OPTION; ②U1将SELECT授予U2; ③DBA又执行GRANT SELECT ON Employee TO U2; ④REVOKE SELECT ON Employee FROM U1 CASCADE; 操作④后U2是否还有SELECT权限？

A.是，因U2从DBA直接获得，不受U1回收影响
B.否，因CASCADE级联回收U1传播的所有权限
C.是，因U2权限来自DBA，CASCADE只回收U1传播的
D.否，因U2权限依赖U1

满足4条：操作序列场景、选项微差（A与C措辞接近）、合理陷阱（B对应"级联回收一切"误解）、需推理（路径追踪）。

### 反例（软工第四章原第8题）

模块A调用模块B时传递flag，B根据flag决定执行哪段代码。耦合类型是？
A.数据耦合 B.控制耦合 C.特征耦合 D.公共耦合

只满足0条：一句话特征直接套定义，选项无微差，无陷阱，靠"传flag→控制耦合"口诀秒答。需改造为伪代码+陷阱题。

## 七、格式规范

### 选择题文件格式
- **题号直接在题干前**：`1. 题干内容（  ）`（数字编号 + 点 + 空格 + 题干）
- **答题括号**：题干末尾加 `（  ）`（全角括号 + 两个空格），跨行题干括号放在问句末尾而非引导句后
- **多选标注**：题号后加 `（多选）`，如 `16.（多选）下列...（  ）`；单选不加标记
- **★★★标记**：放在题号后、题干前，加空格，如 `3. ★★★ 下列内聚...（  ）`；不写"必考/坑点/计算"等文字
- **无标题、无分隔线**：不写 `### 第X题` 标题，题间不加 `---` 分隔线
- **文件结构**：`# 标题` → `> 说明` → 题目列表 → `## 答案` → 答案行（`1.X 2.X ...`）

### 解析文件格式
- **每题标题**：`### 第 X 题`（"第"后空格、数字两边空格、"题"前空格）
- **答案**：`**答案：X**`（中文冒号）
- **解析**：`**解析：**`
- **点睛金句**：`> ` 引用块
- **题间分隔**：`---` 分隔线
- **答案速查**：文末 `## 答案速查` + 表格（`| 题号 | 1 | 2 | ... |`、`| 答案 | X | Y | ... |`）

### 跨行题干括号位置（易错）
题干跨多行时（含代码块、任务表、场景描述），括号放在**问句末尾**，不放引导句后：
- 正例：`阅读以下伪代码：` + 代码块 + `下列说法正确的是：（  ）`
- 反例：`阅读以下伪代码：（  ）` + 代码块 + `下列说法正确的是：`

- 题目：`数据库/复习题/选择题/第四章_数据库安全性_选择题.md`
- 解析：`数据库/复习题/选择题/第四章_数据库安全性_选择题解析.md`

## 八、生成时的检查清单

生成每道深度题后，逐条自检：

- [ ] 题干是否有具体场景/代码/数据（不是纯概念提问）？
- [ ] 正确项与干扰项是否措辞接近、差别在机制？
- [ ] 至少一个干扰项是否对应常见误解或相邻概念？
- [ ] 能否仅靠背诵答出？若能，则不够深
- [ ] 满足4条中至少2条？若不足，改造或降为基础题
- [ ] 格式检查：题号 `X.` 开头？题干末尾有 `（  ）`？多选标注在题号后？★★★后有空格？
