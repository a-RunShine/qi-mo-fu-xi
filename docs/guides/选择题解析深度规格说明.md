# 选择题解析深度规格说明（解析生成规范）

本规范用于约束 AI 生成选择题解析时的质量标准。所有选择题解析生成任务必须遵守。

参考样本：
- 数据库：`数据库/复习题/选择题/第四章_数据库安全性_选择题解析.md`
- 软件工程：`软件工程/复习题/选择题/第四章_软件设计_选择题解析.md`

适用范围：数据库 + 软件工程各章 `*_选择题解析.md`。

---

## 一、两类题的解析形式

选择题分基础题（纯记忆型）和深度题（理解应用型），两类题定义见 `选择题深度规格说明.md`。**解析形式随题类不同**：

### 基础题解析（精简式）

基础题靠背诵即可作答，解析**省略正文**，只保留答案 + 金句：

```
### 第 X 题

**答案：X**

> [金句：口诀/边界提醒]

---
```

**正例（软工第四章第3题）**：
> ### 第3题
> **答案：D**
> > 内聚7级从低到高：偶然→逻辑→时间→过程→通信→顺序→功能。功能内聚是最高……

### 深度题解析（完整四段式）

深度题需讲透机制，必须完整四段式：

```
### 第 X 题

**答案：X**

**解析：**

[正文]

> [金句]

---
```

**判断依据**：若该题在选择题规范里是深度题（满足4条中≥2条），解析按完整四段式；若是基础题，按精简式。

---

## 二、文档结构骨架（强制）

- 文档标题：`# 第X章 章名 — 选择题解析`（数据库带"—"；软工现有文档无"—"，新生成时按带"—"对齐）
- 顶部可选说明：`> 对应 XXX_选择题.md，每题附详细解析与坑点说明。`（软工样本有，数据库无，可选）
- 顶部分隔：`---`
- 题号格式：`### 第 X 题`（"第""X""题"间有空格；软工现有文档无空格，新生成时按有空格对齐）
- 每题区块：按第一节的基础题/深度题两种形式之一
- 文档结尾：`## 答案汇总`（数据库）或 `## 答案速查`（软工），格式二选一：
  - **行式**：`1.X 2.X 3.X …`（一行排完，适用题量少）
  - **表格**：`| 题号 | 1 | 2 | …`（适用题量多，如17题以上）

**禁止**：省略答案行、把解析并进题干、用 `### 解析` 单独成节、深度题省略正文。

---

## 三、深度解析 vs 肤浅解析对照

| 维度 | 肤浅解析（禁止） | 深度解析（要求） |
|---|---|---|
| 开场 | 直接念选项 | 首句点破"这题考的是 XXX" |
| 逻辑链 | 平铺直叙 | 含多步操作/多层概念时，用路径/列表/对比可视化 |
| 正确项 | 只说"对的" | 说"为什么对"，点出关键机制 |
| 错误项 | 只说"不选" | 说"错在哪个具体点" |
| 陷阱 | 不提 | 至少一处点出干扰项对应何种误解 |
| 瑕疵题 | 回避或硬选一个 | 诚实标注并解释取舍 |
| 收尾 | 无 | `>` 金句：口诀/类比/边界提醒 |
| 篇幅 | 2-3 行 | 深度题 8-15 行；基础题金句 1-3 行 |

## 四、正文 5 条强制标准（深度题）

### 1. 首句点破考点

深度题解析正文第一句必须开门见山点明这道题考什么机制/概念，不绕弯、不先念题。基础题因省略正文，由金句承担点题职责。

**正例（第四章第1题）**：
> 这题考的是 **CASCADE 级联回收的范围**。

**正例（软工第四章第1题）**：
> 这题考的是**总体设计与详细设计的阶段边界**——总体设计管"结构"，详细设计管"过程"。

**反例**：
> 题目说 DBA 执行了四个操作……（先复述题干，信息量低）

注：样本中部分深度题首句直接进入机制描述（如"审计的本质是事后记录"），也算点破考点，不强制句式，但必须开门见山点主题。

### 2. 理清逻辑链（含多步操作/多层概念时）

当题目含多步操作、多层概念关系时，用路径分解、有序列表、对比表格等方式可视化，让读者一眼看清结构。纯概念辨析题（单一机制对比）可省，直接逐选项展开。

**正例（第四章第1题）**——理清 U2 获得权限的两条路径：
> - 路径一：DBA → U1（带 WITH GRANT OPTION）→ U2（U1 转授的）
> - 路径二：DBA → U2（DBA 直接授的）

**正例（第四章第7题）**——列出密级排序后逐级判断：
> 四个等级从高到低：绝密 > 机密 > 秘密 > 公开。
> 用户许可证是"机密"，能读的数据密级必须 ≤ 机密……

### 3. 逐选项审判

每个选项都要交代：正确项说"为什么对"（点出关键机制），错误项说"错在哪个具体点"。**禁止只说"不选 B"而不说 B 错在哪。** 对强干扰项可标注"（强干扰项）"，提示读者重点关注。

**正例（第四章第3题）**：
> B 错：INSERT 权限在教材口径下是**表级**的，不细分到列。列级权限主要用于 SELECT 和 UPDATE。
> C 错：SELECT 是**对象权限**，授权时用 `WITH GRANT OPTION`；`WITH ADMIN OPTION` 是给**角色**授权时用的，张冠李戴。

**正例（软工第四章第17题）**——强干扰项标注：
> B 错（强干扰项）：抽象是"抽出本质特征隐藏细节"，方向看似对但不精确——抽象关注"隐藏复杂度、暴露接口"，而信息隐藏更具体地强调"内部数据结构不可见"。

**反例**：
> B 不对，C 不对，选 A。（没说错在哪）

### 4. 点穿陷阱

在逐选项审判基础上，**至少对一处干扰项额外点出它对应何种常见误解或相邻概念混淆**，让读者"恍然大悟"——原来这个错项是这么骗人的。

与第3条的区别：第3条说"错在哪个点"，第4条进一步揭示"这个错项为何有迷惑性、对应什么误解"。两者可合并表达，但陷阱揭示这一层不能缺。

**正例（第四章第8题）**——揭示两 OPTION 的迷惑机制：
> 对象权限用 GRANT OPTION，回收会**级联**（砍树枝）；角色用 ADMIN OPTION，回收**不级联**（各管各的）。这是两者的关键区别。

**正例（第四章第1题）**——点穿 A 项"理由太绝对"的陷阱：
> A 的结论对（"是"），但理由说"不受 U1 被回收的影响"太绝对，U2 通过 U1 的那条权限确实被回收了。

### 5. 瑕疵题诚实标注

遇到双正确答案、题干歧义、教材口径分歧等瑕疵，**诚实标注**并解释为何选其中一个作为标准答案，不掩饰、不硬掰。

**正例（第四章第3题）**：
> D 语法本身也合法（PUBLIC 可作为 REVOKE 的对象），本题存在 A、D 双正确答案的瑕疵。A 更能体现 `GRANT ALL PRIVILEGES ... TO PUBLIC` 的标准授权写法，故标准答案为 A。

**正例（软工第四章第5题）**——教材口径分歧：
> 以张海藩教材为准，部分教材口径不同。

## 五、收尾金句（每题都要有）

每题（含基础题）末尾用 `>` 引用块给一句凝练总结，形式三选一或组合：

1. **口诀**：把机制压缩成顺口溜，便于记忆
   - 第四章第7题：`记住口诀：只能往下看，不能往上看。`
   - 软工第四章第3题：`口诀"偶逻时过通顺功"，最后一个"功"是最高。`

2. **生活化类比**：用日常事物具象抽象概念，是"恍然大悟"的关键手法
   - 审计 = 监控摄像头（能录像追查，不能当场拦人）
   - 角色 = 会员卡（R1 加权限，持卡人自动享受）
   - 视图 = 给表戴面具（只露该露的列）
   - 加密 = 最后一道防线（偷走了也是乱码）
   - CASCADE = 砍树枝（只砍被回收者传播的）
   - MAC = 图书馆借阅规则（你说不算，规定说了算）
   - 顺序内聚 = 流水线（软工）
   - 控制耦合 = 指挥棒（软工）
   - 信息隐藏 = 模块只暴露"做什么"，藏起"怎么做"和内部数据（软工）

3. **边界提醒**：点出易混边界、相邻概念的分界线
   - 第四章第3题：`区分两个 OPTION：WITH GRANT OPTION 搭配对象权限，WITH ADMIN OPTION 搭配角色。记反了就掉坑。`

**可选延伸**：金句后可补一句相邻知识，把单题考点扩成知识面。
- 第四章第7题金句后补：`补充：MAC 还有对称的向上写规则——只能往等于或高于自己许可证级别的客体写，防止高密级数据流向低密级主体。`

**跨题关联（可选）**：与同章相邻题对照时，金句或首句可点出对照关系，强化记忆。
- 软工第四章第2题首句：`这题与第1题互补，考的还是阶段边界，但方向反了。`
- 软工第四章第2题金句：`与第1题对照记忆：总体设计=划模块+定接口，详细设计=写算法+定内部逻辑。`

## 六、风格与篇幅

- **语言**：中文口语化，像老师讲课；关键机制/术语用 **粗体** 突出
- **驱动方式**：以"为什么"驱动，不堆定义；先讲机制再下结论
- **篇幅**：深度题正文 8-15 行（不含金句）；基础题无正文，金句 1-3 行
- **PDF 兼容**：不用 `⭐⊂✓✗` 等非常用 Unicode，用 `【重点】`/`是...的子集`/`正确`/`错误` 替代；箭头 `→` 在口诀序列中可用（软工样本已用），其他特殊符号避免
- **代码**：SQL/伪代码用反引号行内代码，不整段 fenced block

## 七、学科适配

| 学科 | 解析侧重 | 类比素材库（样本实际出现） |
|---|---|---|
| 数据库 | 操作链追踪（授权路径、级联回收）、机制辨析（DAC/MAC、两 OPTION、三系统权限） | 会员卡、面具、摄像头、树枝、防线、图书馆借阅 |
| 软件工程 | 概念辨析（内聚耦合类型、覆盖级别）、场景推理（过程模型选用、风险策略） | 流水线（顺序内聚）、工人处理零件（通信内聚）、指挥棒（控制耦合）、管孩子/被爹叫（扇出扇入）、信息隐藏=接口封装 |

两学科共用：首句点破、逐选项审判、金句收尾、瑕疵诚实标注等结构标准不变。软工其他章节类比素材：传话游戏（需求分析）、体检项目（测试）、保险（风险策略）。

## 八、生成时的检查清单

### 深度题自检

- [ ] 首句是否点破考点（不是先念题干）？
- [ ] 含多步操作/多层概念时，是否用路径/列表/对比可视化？（纯概念辨析题可省）
- [ ] 正确项是否说了"为什么对"（点出关键机制）？
- [ ] 每个错误项是否说了"错在哪个具体点"（不是只说"不选"）？
- [ ] 是否至少点穿一处陷阱（揭示对应何种误解/相邻概念，而非仅说错在哪）？
- [ ] 瑕疵题是否诚实标注并解释取舍？
- [ ] 是否有 `>` 金句（口诀/类比/边界提醒）？
- [ ] 篇幅是否 8-15 行，口语化 + 粗体关键词？
- [ ] 是否 PDF 兼容（无 `⭐⊂✓✗` 等非常用 Unicode）？

### 基础题自检

- [ ] 是否省略正文，只保留答案 + 金句？
- [ ] 金句是否点出口诀/边界提醒（承担点题职责）？
- [ ] 篇幅是否 1-3 行？

### 全篇自检

- [ ] 文档结尾是否有答案汇总/速查（行式或表格）？
- [ ] 题号格式、标题格式是否统一？
