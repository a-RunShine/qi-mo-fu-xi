## 命名规范差异说明

Java刷题速成/ 的 `00_总览` / `01_xxx` / `...` 数字编号体系与 CLAUDE.md 强制规范的「第X章_章节名_类型」不一致。

- **是否需要统一?**
  - 优点:统一后 agent 处理时不需要特例
  - 缺点:Java刷题速成的编号是有意义的(00=总览,01-06=主题,07=实战),改中文章号会破坏语义
- **建议**:**不统一**,Java刷题速成作为特殊学科,CLAUDE.md 第 2 节单独标记「数字编号体系」

## 后续迁移触发条件

如果以下条件之一出现,触发本方案执行:

1. Java 学科被正式纳入期末复习范围(目前是速成独立项)
2. 数据库期末突击/ 出现第二份试卷(代表成为常驻产物)
3. 复习PDF/ 超过 10 个 zip(归档膨胀,需要轮转)
4. 出现新的"野生"产物(目前是这 4 类边界外的新文件)

触发后:

1. 评估新产物是否纳入命名规范
2. 若纳入:补 CLAUDE.md 第 2/3 节 + 重新生成 CLAUDE.md
3. 若不纳入:在本文件加一行,保持只读归档

## 备份与回滚

野生产物的所有修改在执行前必须:

- 复制到 `/tmp/wild_product_backup_$(date +%Y%m%d)/`
- 记录 `git log --oneline -5` 当前 commit hash
- 若失败,`git checkout -- <file>` 回滚
