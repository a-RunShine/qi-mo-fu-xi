## 一、问题表现

用户反馈:**04 最后一页的练习建议中,markdown 语法被暴露出来了**。

用 PyMuPDF 提取 04 PDF 末尾实际文本,发现:
```
练习建议
>             ← 字面 `>` 暴露
- Arrays 必做:...   ← 字面 `-` 暴露(应渲染成 `•`)
- Collections 必做:...
...
下一步:打开 ...
```

**暴露了 2 类 markdown 符号**:
1. 引用块符号 `>`(脚本处理空 `>` 行时,当普通段落输出)
2. 列表项短横线 `-`(脚本引用块内嵌套 `-` 时,不转 `•`)

## 二、根本原因分析

### 2.1 脚本行为

`convert_md_to_pdf.py` 的引用块处理(第 532-536 行):
```python
if line.startswith("> "):
    pdf._render_blockquote(line[2:])
    i += 1
    continue
```

**问题**:
- 只处理 `> `(带空格)开头的行
- 单独 `>` 行(无空格)被当普通段落,走 `add_paragraph`,**字面输出 `>`**
- `> - xxx` 这种"引用块 + 列表"嵌套,`line[2:]` 去掉 `> ` 后是 `- xxx`,**保留字面 `-`**,不转 `•`**

### 2.2 触发条件

必须**同时满足**:
1. 文档用了 `> **练习建议**` 这种**引用块**结构
2. 引用块内嵌**多个 `- xxx` 列表项**
3. 引用块内嵌**空行**(单独 `>` 行)

`04_常用工具类.md` 完全满足 → 暴露严重。

## 三、批量修复(6 篇 .md + 6 篇 PDF)

### 3.1 修复策略

把 `> **练习建议** + > - xxx + > **下一步**` 整段改成:
```
## 练习建议
- xxx
- xxx
...

## 下一步
xxx
```

**好处**:
- `##` 标题正常渲染(无 `**` 暴露)
- `-` 列表项正常渲染成 `•`
- 没有 `>` 引用块嵌套,不会有 `>` 暴露

### 3.2 兼容两种格式

`> **练习建议**` 后接**空 `>` 行**(00/03/05)还是**直接 `-` 列表**(01/02/04/06),regex 要兼容:
```python
PATTERN = re.compile(
    r"^>\s*\*\*练习建议\*\*\s*\n"
    r"(?:^>\s*\n)*"            # 0+ 个空 `>` 行
    r"((?:^>\s*-\s+.*\n?)+)",  # 多行 `> - xxx`
    re.MULTILINE
)
```

## 四、给所有 PDF 文档的强制规则(累加)

### 4.1 不要用 `>` 引用块 + `-` 列表嵌套

**规则**:**章节末尾的"练习建议"/"下一步"/"金句"等,必须用 `##` 或 `###` 标题 + `-` 列表,不要用 `> **标题**` + `> - xxx` 嵌套**。

```markdown
错误(暴露 `>` 和 `-`):
> **练习建议**
>
> - xxx
> - xxx

正确(正常渲染):
## 练习建议
- xxx
- xxx
```

### 4.2 不要用 `> **xxx**:` 单独引用块

**规则**:**单行的"下一步"/"金句"等,用 `## 下一步` 标题 + 段落,不要用 `> **下一步**:xxx` 引用块**。

### 4.3 空 `>` 行会被字面输出

**规则**:**任何 `> ` 行后面**必须**跟非空内容,不要单独写一行 `>`**。或者**用 `##` 标题代替引用块**。
