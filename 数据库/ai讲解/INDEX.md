## 跨章逻辑主线

**数据库系统 = 4 大 DBMS 任务**（第一章点题，第十一章收尾呼应）：

```
第一章 绪论 —— 搭起总框架
   数据库 = DB + DBMS + 应用程序 + DBA
   4 大任务 = 安全性 + 完整性 + 恢复 + 并发控制
   3 级模式 + 2 级映像 = 解决"独立性"
       ↓
┌──────────────────────────────────────────────┐
│  4 大 DBMS 任务（核心机制）                       │
│                                                │
│  第 4 章 安全性  ─── 管"门"（谁能用）            │
│  第 5 章 完整性  ─── 管"货"（数据对不对）        │
│  第 10 章 恢复  ─── 故障后怎么救                 │
│  第 11 章 并发控制 ─── 多用户怎么不打架          │
└──────────────────────────────────────────────┘
       ↓
第 7 章 数据库设计 —— 怎么从需求到数据库
   6 步流水线：需求 → 概念 → 逻辑 → 物理 → 实施 → 维护
   （维护阶段 = 4 大任务在运行期的持续应用）
       ↓
第 8 章 数据库编程 —— 怎么在 C/Java 里用 SQL
   嵌入式 SQL + SQLCA + 主变量 + 游标
   （一个嵌入式 SQL 程序 = 一个或多个事务）
       ↓
第十一章末呼应 ── 全数据库系统"画龙点睛"
```

## 推荐复习路径

1. **先看第 1 章** —— 搭起总框架，知道数据库是什么、4 大任务是什么
2. **再看第 4/5 章** —— 4 大任务中最"日常"的两项：谁能用、数据对不对
3. **第 7 章** —— 把"现实世界需求"翻译成"数据库"的 6 步流程
4. **第 8 章** —— 学会在 C/Java 程序里调用 SQL
5. **第 10 章** —— 故障恢复（ACID 是核心）
6. **第 11 章** —— 并发控制（3 不一致、3 协议、2PL）
