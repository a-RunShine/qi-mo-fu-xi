# 大题解题方法 —— AI 讲解（应试突击版）

## 根本矛盾

大题考的不是概念背诵，是**套路应用**——给你 U、F、关系，给你题干描述，按固定模板求解。掌握每个题型的"看到什么 → 用什么方法 → 写出什么格式"，大题就能拿满。

---

## 板块 1：关系理论大题套路

### 1.1 函数依赖三剑客

| 概念 | 定义 | 应试判断 |
|------|------|----------|
| **平凡** | X→Y 且 Y属于X | X 的子集，废话，没意义 |
| **非平凡** | X→Y 且 Y不属于X | 有意义的依赖，考试默认讨论这个 |
| **完全** | X→Y，去掉 X 任何真子集都不能推出 Y | 左侧不能再缩 |
| **部分** | X→Y，X 的真子集也能推出 Y | 例：A→C 且 AB→C，C 部分依赖 AB |

> **应试口诀**："非平凡才有意义，完全依赖要求严"

---

### 1.2 求候选码两步法

**Step 1：L/R/LR 类划分**

| 类别 | 特征 | 结论 |
|------|------|------|
| **L 类** | 只在 → 左侧出现 | 必在候选码中 |
| **R 类** | 只在 → 右侧出现 | 绝不在候选码中 |
| **LR 类** | 左右都出现 | 待定，逐个试 |
| **N 类** | 根本没出现 | 必在候选码中 |

**Step 2：闭包验证**
- 设 X 为当前属性组，`X+ = X`
- 反复用 F 中的 FD 往 X+ 里加属性，直到不能再扩
- 若 `X+ = U`，则 X 是候选码

**例题**：U={A,B,C,D,E,G}，F={AB→C, CD→E, E→A, A→G}
- L 类：B、D（只在左侧）→ 必在码中
- LR 类：A、C、E（左右都出现）→ 待定
- R 类：G（只在右侧）→ 不在码中
- 先试 BD：BD+ = BD（推不出其他）→ 不行
- 试 BDA：BDA+ = ABCDEFG → 候选码 {BDA}
- 试 BDC：BDC+ = ABCDEFG → 候选码 {BDC}
- 试 BDE：BDE+ = ABCDEFG → 候选码 {BDE}

**概念区分**：
- **候选码**：极小超码（去掉任意属性就不能推出全部）
- **超码**：能推出全部属性的任意属性组
- **主码**：多个候选码中挑一个
- **主属性**：出现在某个候选码中的属性
- **非主属性**：不在任何候选码中的属性
- **外码**：本关系不是码，但它是另一个关系的码
- **全码**：整个属性组都是码

> **应试口诀**："L 类必在码中，LR 类逐个试；闭包到 U 才算赢"

---

### 1.3 范式四步判断

| 步骤 | 检查什么 | 典型反例 |
|------|----------|----------|
| 1NF | 属性不可再分（默认满足） | — |
| 2NF | 非主属性完全依赖候选码 | A→D（A 是半截） |
| 3NF | 非主属性间无传递依赖 | D→E（桥梁） |
| BCNF | 所有依赖左侧都含码 | A→C（A 不含码） |

**2NF vs BCNF 核心区别**：
- **2NF**：只看非主属性的左侧
- **BCNF**：看所有依赖的左侧

> **应试口诀**："2NF 看非主，3NF 拦传递，BCNF 管得宽"

### 1.3b BCNF 分解大题套路

BCNF 分解是期末高频大题——先判断是否满足 BCNF，若不满足则分解。

**分解方法**：
1. 找出违反 BCNF 的 FD X→Y（左侧 X 不含码）
2. 将原关系 R 分解为 R₁(XY) 和 R₂(R−Y)
3. 对 R₁ 和 R₂ 递归判断 BCNF，直到全部满足

**例题**：R(ABC)，F={A→B, B→C}，候选码为 A
- B→C 左侧 B 不含码 → 违反 BCNF
- 分解：R₁(B,C) 和 R₂(A,B)
- R₁：候选码 B，B→C 左侧 B 含码 ✓
- R₂：候选码 A，A→B 左侧 A 含码 ✓
- 两者都满足 BCNF，分解完成

> **应试口诀**："BCNF 不满足，找反例 FD 拆成两半，递归直到全满足"

---

### 1.4 最小函数依赖集三步法（总览）

最小函数依赖集 Fmin 是 F 的"最简版本"——没有冗余依赖、左侧没有多余属性。三步走：

```
Step 1：拆分右侧（每个 FD 右侧只留一个属性）
Step 2：去除自身求闭包（去掉冗余 FD）
Step 3：左侧最小化（去掉左侧多余属性）
```

**综合例题**：F={C→A, CG→BD, CE→A, ACD→B}

| 步骤 | 操作 | 结果 |
|------|------|------|
| Step 1 | CG→BD 拆为 CG→B 和 CG→D | F={C→A, CG→B, CG→D, CE→A, ACD→B} |
| Step 2 | (C)+=C，保留 C→A；(CG)+=CGADB，去掉 CG→B；(CG)+=CGA，保留 CG→D；(CE)+=CEA，去掉 CE→A；(ACD)+=ACD，保留 ACD→B | F={C→A, CG→D, ACD→B} |
| Step 3 | ACD→B：C→A，去掉 A 得 CD→B 仍成立 | Fmin={C→A, CG→D, CD→B} |

> **应试口诀**："拆右、去自、压左"

---

### 1.4a 拆分右侧

**规则**：将每个 FD 的右侧拆成单个属性。即 X→A₁A₂...An 拆为 X→A₁, X→A₂, ..., X→An。

**例题**：F={A→BC, D→EF}
- A→BC 拆为 A→B 和 A→C
- D→EF 拆为 D→E 和 D→F
- 结果：F={A→B, A→C, D→E, D→F}

> **应试口诀**："右侧多个属性，拆成一行一个"

---

### 1.4b 去除冗余（去自求闭包）

**规则**：对每个 FD X→Y，从 F 中暂去掉它，求 X+。若 X+ 仍包含 Y，则这个 FD 是冗余的，删除。

**例题**：F={AB→C, B→C}
- 暂去掉 AB→C，求 (AB)+：B→C 可推出 C，所以 (AB)+ 包含 C
- AB→C 是冗余 → 删除
- 结果：Fmin={B→C}

> **应试口诀**："去掉自己求闭包，能推就删"

---

### 1.4c 左侧最小化（压左）

**规则**：对每个 FD X→Y，逐个检查 X 中的属性能否去掉。若去掉后仍能推出 Y，则去掉。

**例题**：F={ABC→D, C→A}
- ABC→D：C→A，所以 A 可由 C 推出，去掉 A 得 BC→D
- 检查 BC→D：B 不能由 C 推出，保留；C 不能由 B 推出，保留
- 结果：Fmin={BC→D, C→A}

> **应试口诀**："左侧从右往左试，能推就去掉"

---

### 1.5 模式分解两大题型

#### 题型 A：判断无损连接（表格法）

| 步骤 | 操作 |
|------|------|
| 1 | 画表格，行=分解关系，列=所有属性 |
| 2 | 关系中有的属性标 aj（j=列号），没有的标 bij（i=行号） |
| 3 | 对每个 FD X→Y，找 X 列同行相同标记，更新 Y 列 |
| 4 | 重复直到某行全 a（无损）或无变化（非无损） |

**例题**：R(ABCDE) 分解为 R₁(CDE)、R₂(AE)、R₃(AB)，F={A→C, B→C, C→D, DE→A, AE→B}

| 属性 | A | B | C | D | E |
|------|---|---|---|---|---|
| R₁ | b₁₁ | b₁₂ | a₃ | a₄ | a₅ |
| R₂ | a₁ | b₂₂ | b₂₃ | b₂₄ | a₅ |
| R₃ | a₁ | a₂ | b₃₃ | b₃₄ | b₃₅ |

反复用 FD 更新表格后，若某行全部变为 a，则无损。

#### 题型 B：分解为 3NF（保持无损+依赖）

| 步骤 | 操作 |
|------|------|
| 1 | 求最小函数依赖集 Fmin |
| 2 | Fmin 中没出现的属性单独成一组 |
| 3 | Fmin 中同左部的依赖合并成一组 |
| 4 | 候选码单独一组（若未出现在前面各组中） |

> **应试口诀**："Fmin 拆无合同，候选码单飞"

## 板块 2：关系语言大题套路

### 2.1 五大运算速查

| 运算 | 符号 | 作用 | 典型场景 |
|------|------|------|----------|
| 选择 | σ | 选行 | "成绩>80" |
| 投影 | π | 选列 | "只显示姓名" |
| 连接 | 连接 | 跨表 | 多表查询 |
| 除 | ÷ | 满足"全部" | "选了所有课的学生" |
| 差 | − | 排除 | "没有不及格的学生" |

**完整例题**：学生表 S(学号,姓名)，选课表 SC(学号,课程号)，课程表 C(课程号,课程名)。求"选了所有课程的学生姓名"。
- 关系代数：`π 姓名 (S 连接 (π 学号,课程号 (SC) ÷ π 课程号 (C)))`
- 分析：先对 SC 投影学号和课程号，对 C 投影课程号，相除得到选了所有课的学生学号，再连接 S 得到姓名

### 2.2 三类典型题模板

**类型 1：常规题（投影+选择+连接）**

```
π 投影列 (σ 选择条件 (R 连接 S 连接 T))
```

从内到外：先连接 → 再选择 → 最后投影。

**类型 2：除运算（题干出现"全部"二字）**

```
π 学号,课程 (选课表) ÷ π 课程 (课程表)
```

套路：先对 A 投影属性组，对 B 投影"全部"属性组，再相除。

**类型 3：差运算（题干出现"没有…低于…"、"至少一个不…"）**

```
π 姓名 (学生表) − π 姓名 (σ 成绩<80 (成绩表))
```

套路：全集 − 满足否定条件的子集。

> **应试口诀**："常规连接选投，全部用除，否定用差"

---

### 2.3 元组关系演算

**基本格式**：`{t | P(t)}` — t 是元组变量，P(t) 是条件公式

**三要素**：
1. `R(t)` — t 属于关系 R
2. `t[i] θ u[j]` — t 的第 i 分量与 u 的第 j 分量比较
3. `t[i] θ c` — t 的第 i 分量与常量 c 比较

**关系代数 ↔ 元组演算转换表**：

| 关系代数 | 元组演算 |
|----------|----------|
| R∪S | {t | R(t) ∨ S(t)} |
| R∩S | {t | R(t) ∧ S(t)} |
| R−S | {t | R(t) ∧ ¬S(t)} |
| R×S | {t[1..n+m] | R(t[1..n]) ∧ S(t[n+1..n+m])} |
| π | 改变 t 的目数 |
| σ | 加 F 条件 |

**解题格式**：
```
S = {t | (量词运算) (元组所属关系 ∧ 条件)}
```

> **应试口诀**："并或交与差非，积加目数变投影"

## 板块 3：事务调度大题套路

### 3.1 调度基本准则

- **一组调度**：包含所有事务指令，保持事务内顺序
- **并行调度**：可串行化（结果与某串行调度一致）

### 3.2 冲突可串行化

**冲突操作**：不同事务对同一数据（一读一写 / 一写一读 / 两写）

**判断方法**：画前驱图
- 节点 = 事务
- 有向边 = 冲突顺序
- **无环 → 可串行化**

**例题**：调度 S：R₁(A) W₂(A) R₂(B) W₁(B) R₁(C) W₂(C)
- 冲突操作分析：
  - R₁(A) 与 W₂(A) 冲突 → T₁ → T₂
  - W₁(B) 与 R₂(B) 冲突 → T₁ → T₂
  - R₁(C) 与 W₂(C) 冲突 → T₁ → T₂
- 前驱图：T₁ → T₂（无环）
- 结论：可串行化，等价于串行调度 T₁→T₂

> **应试口诀**："同数据读写写写，画图无环就 OK"

### 3.3 封锁

| 锁 | 上锁后能做什么 | 其他事务能加 |
|----|----------------|--------------|
| X 锁（写锁） | 读 + 改 | 不能加任何锁 |
| S 锁（读锁） | 只能读 | 只能加 S 锁 |

### 3.4 三级封锁协议

| 级别 | 规则 | 防止什么 |
|------|------|----------|
| 一级 | 写前加 X，事务结束释放 | 丢失修改 |
| 二级 | 一级 + 读前加 S，读完释放 S | + 读脏数据 |
| 三级 | 一级 + 读前加 S，事务结束释放 | + 不可重复读 |

> **应试口诀**："一级防丢失，二级加防脏，三级再加不可重复读"

### 3.5 两段锁协议（2PL）

- **规则**：所有加锁在所有释放之前（加锁段 → 释放段）
- 2PL 是可串行化的**充分条件**（不是必要条件）
- 2PL 可能发生**死锁**
- 三级封锁协议是 2PL 的特例（更严格）

> **应试口诀**："2PL 充分不必要，可能死锁别忘记"

## 板块 4：数据库设计大题套路

### 4.1 E-R 图三要素

| 要素 | 画法 | 说明 |
|------|------|------|
| 实体 | 矩形 □ | 客观存在的对象 |
| 属性 | 椭圆 ○ | 实体的特征 |
| 关系 | 菱形 ◇ | 实体间的联系 |

**联系类型**：1:1 / 1:n / m:n

注意：**实体和关系都可以有属性**。

### 4.2 E-R 图转关系模型（三规则）

| 联系类型 | 转换规则 |
|----------|----------|
| **1:1** | 任意一方加入对方主码作为外码 + 联系属性 |
| **1:n** | 1 方主码加入 n 方作外码 + 联系属性 |
| **m:n** | 联系本身转为一个关系模式 + 双方主码 + 联系属性（码是双方主码的组合） |

**完整例题**：学校有教师（工号,姓名）和课程（课程号,课程名），一个教师可教多门课，一门课只有一个教师（1:n）。
- 教师关系：教师(工号,姓名) — 码：工号
- 课程关系：课程(课程号,课程名,工号) — 码：课程号，外码：工号
- 1 方（教师）的主码工号加入 n 方（课程）作为外码

> **应试口诀**："一对一任选，一对多靠多，多对多独立成关系"

## 易混淆对比总结

| 对比项 | 区别 | 记忆方法 |
|--------|------|----------|
| 最小依赖集 vs 模式分解 | 前者化简 FD，后者拆表 | "先化简再分解" |
| 关系代数 vs 元组演算 | 前者符号运算，后者逻辑公式 | "代数符号化，演算公式化" |
| 一级 vs 二级 vs 三级封锁 | 释放时机不同 | "一级事务结束放写，二级读完放读，三级事务结束放读" |
| 三级封锁 vs 2PL | 三级是 2PL 的特例 | "三级更严，2PL 更灵活" |
| 1:1 vs 1:n vs m:n 转换 | 外码放哪不同 | "一对一到任，一对多到多，多对多独立" |

## 逻辑主线

```
给定场景
    ↓
识别题型
    ↓
┌──────────────────────────────────────────┐
│  关系理论  │  关系语言  │  事务调度  │  数据库设计  │
│  F/U 求码  │  代数/演算  │  前驱图  │  E-R/转换  │
│  判范式    │  三类模板  │  封锁协议  │  三规则  │
│  最小依赖  │            │  2PL      │            │
│  模式分解  │            │           │            │
└──────────────────────────────────────────┘
    ↓
套对应模板
    ↓
写出答案
```

**一句话总结**：大题就是**识别题型 → 套对应模板**——关系理论求码判范式，关系语言代数演算，事务调度画图判冲突，数据库设计画图再转换。
