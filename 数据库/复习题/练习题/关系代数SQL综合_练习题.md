### 板块一：关系代数·基础

**第 1 题**：
- （1）`π 姓名,学号 (σ 所在系='计算机系' ∧ 性别='女' ∧ 入学年份≥2020 (S))`
  - `σ 所在系='计算机系' ∧ 性别='女' ∧ 入学年份≥2020 (S)`：**过滤**（按三个条件合取筛选学生元组）
  - `π 姓名,学号 (...)`：**投影**（从筛选后的元组取出"姓名"和"学号"两列）

**第 2 题**：
- （1）`π 商品名,单价 (σ 城市∈{'北京','上海'} ∧ 2024-01-01 ≤ 下单日期 ≤ 2024-12-31 (K ⋈ H ⋈ D ⋈ P))`
  - 连接顺序：K⋈H（顾客与订单）→ ⋈D（加订单明细）→ ⋈P（加商品）
  - DISTINCT 去重：**关系代数 π 本身是集合语义，自带去重**（同一商品被多人下单只出现一次）；若翻译为 SQL 需显式加 `DISTINCT`：`SELECT DISTINCT P.商品名, P.单价 ...`

**第 3 题**：
- （1）`π 客户号,姓名,账户总数,账户总余额 (C ⋈ (G 客户号 F COUNT(账户号) AS 账户总数, SUM(余额) AS 账户总余额 (A)))`
  - G 分组键：`客户号`（每位客户一个组）
  - F 聚合：`COUNT(账户号)` 统计账户数，`SUM(余额)` 累加余额

**第 4 题**：
- （1）`π 学号,姓名 (S ⋈ (π 学号,课程号 (SC) ÷ π 课程号 (σ 课程名∈{'数据结构','离散数学','操作系统'} (C))))`
- （2）`π 姓名 (S ⋈ (π 学号,课程号 (SC) ÷ π 课程号 (C)))`

### 板块二：关系代数 + SQL 过渡

**第 5 题**：
- （1）思路：先找出"有教授任课的课程号集合"（`π 课程号 (σ 职称='教授' (TC ⋈ T))`），再用该集合 ⋈ **所有任课信息**（不限职称），得到完整任课清单
  - 关系代数表达式：
    ```
    π 课程号,课程名,姓名,教室 (
      (π 课程号 (σ 职称='教授' (TC ⋈ T)))
      ⋈ TC ⋈ T ⋈ C
    )
    ```
  - **易错点**：如果直接写 `π 课程号,课程名,姓名,教室 (σ 职称='教授' (C ⋈ TC ⋈ T))`，会把"教授+讲师混合任课"的课程中"讲师"那行过滤掉，导致输出不全。正确做法是"**先筛课程号，再 ⋈ 所有任课行**"——这是"σ 在 ⋈ 之前"原则的反例：不能让 σ 提前吃掉非目标行。

**第 6 题**：
- （1）
  ```sql
  SELECT S.姓名, AVG(SC.成绩) AS 平均成绩
  FROM S JOIN SC ON S.学号 = SC.学号
  WHERE S.所在系 = '计算机系' AND SC.学期 = '2023秋'
  GROUP BY S.学号, S.姓名
  ORDER BY 平均成绩 DESC;
  ```

### 板块三：SQL·提高

**第 7 题**：
- （1）NOT EXISTS 双层嵌套：
  ```sql
  SELECT 姓名 FROM S
  WHERE NOT EXISTS (
    SELECT 1 FROM C
    WHERE C.所属系 = '计算机系'
      AND NOT EXISTS (
        SELECT 1 FROM SC
        WHERE SC.学号 = S.学号 AND SC.课程号 = C.课程号
      )
  );
  ```
- （2）**改用 MIN/MAX 聚合（推荐，比双重否定更直观）**：
  ```sql
  SELECT 姓名 FROM S
  WHERE 姓名 <> '张三'
    AND 学号 IN (
      SELECT 学号 FROM SC
      GROUP BY 学号
      HAVING MIN(成绩) > (
        SELECT MAX(SC2.成绩) FROM SC SC2
        JOIN S S3 ON SC2.学号 = S3.学号
        WHERE S3.姓名 = '张三'
      )
    );
  ```
  - 语义解读：每位学生的 `MIN(成绩) > 张三的 MAX(成绩)`，即"该生最低分都比张三最高分高"——等价于"该生每门课都 > 张三每门课"
  - **陷阱提示**：原题中"IN + > ALL"是**错误**写法，IN 是存在性语义，只要求"至少一门课 > ALL(张三成绩)"，并非"每门课都 >"。正确改法为 MIN/MAX（推荐）或 NOT EXISTS 双重否定。

**第 8 题**：
- （1）相关子查询（"每门课都高于该课均分"，用 NOT EXISTS 表达"不存在一门课成绩 ≤ 该课均分"）：
  ```sql
  SELECT 学号 FROM S
  WHERE NOT EXISTS (
    SELECT 1 FROM SC SC1
    WHERE SC1.学号 = S.学号
      AND SC1.成绩 <= (
        SELECT AVG(SC2.成绩) FROM SC SC2
        WHERE SC2.课程号 = SC1.课程号
      )
  );
  ```
- （2）NOT EXISTS 相关（"没有挂过任何课"= "不存在一门课成绩 < 60"）：
  ```sql
  SELECT 姓名 FROM S
  WHERE NOT EXISTS (
    SELECT 1 FROM SC
    WHERE SC.学号 = S.学号 AND SC.成绩 < 60
  );
  ```

**第 9 题**：
- （1）
  ```sql
  SELECT C.姓名, SUM(A.余额) AS 总余额
  FROM C JOIN A ON C.客户号 = A.客户号
  GROUP BY C.客户号, C.姓名
  HAVING SUM(A.余额) > 100000
  ORDER BY 总余额 DESC;
  ```
- （2）
  ```sql
  SELECT C.姓名, C.城市, COUNT(DISTINCT A.开户行号) AS 支行数
  FROM C JOIN A ON C.客户号 = A.客户号
  GROUP BY C.客户号, C.姓名, C.城市
  HAVING COUNT(DISTINCT A.开户行号) >= 3
  ORDER BY 支行数 DESC;
  ```

**第 10 题**：
- （1）
  ```sql
  SELECT P.姓名, COUNT(DISTINCT B.航班号) AS 订票次数
  FROM P JOIN B ON P.乘客号 = B.乘客号
  GROUP BY P.乘客号, P.姓名
  HAVING COUNT(DISTINCT B.航班号) >= 3
  ORDER BY 订票次数 DESC, P.姓名 ASC;
  ```
- （2）
  ```sql
  SELECT P.乘客号, P.姓名,
         COALESCE(SUM(B.票数 * F.票价), 0) AS 累计金额
  FROM P
  LEFT JOIN B ON P.乘客号 = B.乘客号
  LEFT JOIN F ON B.航班号 = F.航班号
  GROUP BY P.乘客号, P.姓名
  ORDER BY P.乘客号;
  ```
