## 八、案例分析题

26. 某学校数据库中有 Student(Sno, Sname, Sdept)、Course(Cno, Cname, Credit)、SC(Sno, Cno, Grade) 三张表。业务规则：学生必须属于一个学院（Sdept 非空）、学分 1-6、学生只能选本校课程、课程取消时成绩自动删除。请回答以上规则分别对应哪种完整性约束，并写出满足所有规则的 CREATE TABLE 语句。

---

## 答案

### 选择题
1.B 2.B 3.B 4.B 5.B 6.D 7.B 8.B

### 填空题
9.参照 10.唯一，非空 11.外码，被参照 12.级联删除，置空 13.单个属性，一行中的多个属性

### 简答题（要点）
14. 正确性：数据符合现实语义。相容性：同一对象在不同表中一致。例子：学生表显示张三 20 岁，成绩表显示张三 1955 年出生——各自"正确"但"不相容"。

15. ①定义约束→②检查方法→③违约处理。顺序不能反：没定义约束就无法检查，没检查就无法处理。

16. 级联和置空是"父表变动→子表跟随"的链式反应。子表自己造引用失败时，没有"父"可以跟随，只能拒绝。

### 应用题（要点）
17. CREATE TABLE Student (Sno CHAR(9) PRIMARY KEY, Sname CHAR(20) UNIQUE, Sage INT CHECK (Sage>=15 AND Sage<=60), Ssex CHAR(1) CHECK (Ssex IN ('M','F')));

18. CREATE TABLE SC (Sno CHAR(9), Cno CHAR(4), Grade INT CHECK (Grade>=0 AND Grade<=100), PRIMARY KEY (Sno,Cno), FOREIGN KEY (Sno) REFERENCES Student(Sno) ON DELETE CASCADE, FOREIGN KEY (Cno) REFERENCES Course(Cno) ON DELETE CASCADE);

19. 应使用 CREATE ASSERTION 或触发器：
```sql
CREATE ASSERTION male_not_obstetrics CHECK (NOT EXISTS (SELECT * FROM SC, Student, Course WHERE SC.Sno=Student.Sno AND SC.Cno=Course.Cno AND Student.Ssex='M' AND Course.Cname='妇产科学'));
```

### 分析题（要点）
20. (1) 实体完整性，拒绝。(2) 参照完整性，拒绝。(3) 用户定义完整性，拒绝。(4) 参照完整性，根据违约策略处理。

21. (1) 级联删除 SC 中相关记录。(2) 级联修改 SC 中 Sno='S002' 为 'S100'。(3) 拒绝，Sno='S999' 在 Student 中不存在。

### 辨析题
22. 错误。实体完整性要求主码唯一且非空。

23. 错误。属性级 CHECK 约束单列；元组级 CHECK 约束多列间关系，本质不同。

24. 错误。外码可以为空，非空时必须引用被参照表中存在的主码值。

### 纠错题
25. (1) CHECK (Ssex='M' OR 'F') 语法错误，应改为 CHECK (Ssex IN ('M','F'))。
   (2) PRIMARY KEY 和 FOREIGN KEY 缺括号：PRIMARY KEY (Sno,Cno)，FOREIGN KEY (Sno) REFERENCES Student(Sno)。

### 案例分析题（要点）
26. Sdept 非空→用户定义完整性；学分 1-6→用户定义完整性；只能选本校课程→参照完整性；课程取消自动删成绩→级联删除。
