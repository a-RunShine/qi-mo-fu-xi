## 四、程序分析题

17. 分析以下嵌入式 SQL 程序片段的执行流程：

```c
EXEC SQL BEGIN DECLARE SECTION;
char sno[9]; char sname[20]; int sage;
EXEC SQL END DECLARE SECTION;
EXEC SQL INCLUDE SQLCA;
int main() {
    EXEC SQL CONNECT TO testdb USER 'user1';
    EXEC SQL DECLARE SX CURSOR FOR SELECT Sno, Sname, Sage FROM Student WHERE Sage > 20;
    EXEC SQL OPEN SX;
    for (;;) {
        EXEC SQL FETCH SX INTO :sno, :sname, :sage;
        if (SQLCA.SQLCODE != 0) break;
        printf("%s %s %d\n", sno, sname, sage);
    }
    EXEC SQL CLOSE SX; EXEC SQL COMMIT WORK; EXEC SQL DISCONNECT;
    return 0;
}
```

请回答：(1) sno、sname、sage 是什么变量？冒号的作用？(2) SQLCA.SQLCODE 的作用？(3) 如果 Sage>20 的记录有 0 条，程序如何执行？

18. 以下程序通过游标逐行更新学生年龄，请分析性能问题及优化方案：

```c
EXEC SQL DECLARE SX CURSOR FOR SELECT Sno FROM Student WHERE Sdept = 'CS';
EXEC SQL OPEN SX;
for (;;) {
    EXEC SQL FETCH SX INTO :sno;
    if (SQLCA.SQLCODE != 0) break;
    EXEC SQL UPDATE Student SET Sage = 20 WHERE Sno = :sno;
}
EXEC SQL CLOSE SX; EXEC SQL COMMIT WORK;
```

## 答案

### 选择题
1.B 2.A 3.A 4.C 5.A 6.C 7.B 8.B

### 填空题
9.SQLCA 10.成功，失败/异常 11.输入，输出 12.整型 13.OPEN，CLOSE

### 简答题（要点）
14. SQL 面向集合（一次返回多条），主语言面向记录（一次处理一条）。游标作为中间机制逐条提供结果。

15. SQLCA：传递执行状态。主变量：双向传值。游标：逐条获取结果集。协作：SQLCA 检查状态→主变量提供参数→游标获取结果→主变量接收数据。

16. 主变量无法区分"值为 0"和"值为 NULL"。指示变量通过 <0 表示 NULL，=0 正常，>0 截断。

### 程序分析题（要点）
17. (1) 主变量。冒号标识这是主语言变量。(2) 检查 SQL 执行状态，!=0 表示无数据或出错。(3) 第一次 FETCH 时 SQLCODE=100，直接 break。

18. 每次 FETCH 后执行一次 UPDATE，N 条记录 N 次 SQL 调用，性能差。应使用一条 UPDATE 语句或 WHERE CURRENT OF。

### 应用题（要点）
19-20. 见标准嵌入式 SQL 程序模板。

### 辨析题
21. 错误。EXEC SQL 是给预编译器看的。
22. 错误。游标是 SQL 系统的逻辑标记，不是 C 指针。
23. 错误。SQLCODE=100 表示无数据，属于正常情况。

### 纠错题
24. (1) FETCH INTO :sno, :sname。(2) WHERE Sno = :sno。

### 案例分析题（要点）
25. (1) 方案 A：一条 SQL，性能好。方案 B：可逐行复杂处理，性能差。(2) 需逐行业务逻辑判断时。(3) 方案 A 更好，一条 UPDATE 由 DBMS 优化执行。

26. (1) <0=NULL, =0=正常, >0=截断。(2) int 类型无法区分 0 和 NULL。(3) 不会，内连接只返回匹配记录。
