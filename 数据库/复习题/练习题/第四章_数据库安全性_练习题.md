## 八、案例分析题

25. 某军事数据库系统存储了不同密级的作战数据。系统要求：
   - 士兵只能查看"公开"和"内部"级别的数据
   - 军官可以查看"公开"、"内部"和"机密"级别的数据
   - 任何人不能将自己的数据访问权限授予他人
   - 数据访问权限由系统根据用户级别自动控制

   请回答：
   (1) 该系统应该采用 DAC 还是 MAC？为什么？
   (2) 如果采用 DAC 模型，能否满足"任何人不能将自己的权限授予他人"的要求？为什么？
   (3) 简述 MAC 模型中的"向下读、向上写"原则在该场景中如何体现。

## 答案

### 选择题
1.B 2.C 3.B 4.B 5.B 6.B 7.C 8.B

### 填空题
9.存取控制，数据加密 10.WITH GRANT OPTION，WITH ADMIN OPTION 11.权限，授权 12.CONNECT 13.敏感度标记，许可证级别

### 简答题（要点）
14. DAC：数据所有者自主决定权限，灵活但易扩散，适用普通企业。MAC：系统根据安全标记强制控制，用户不能改变，适用军事、政府系统。

15. CASCADE：级联回收传播的权限。RESTRICT：有传播则拒绝回收。RESTRICT 不是"部分回收"，而是"有传播就不让收"。

16. 循环授权如 A→B→C→A，若 A 权限被收回，理论上 B、C 也应消失，但 A 又从 C 获得权限，形成死循环。SQL 一刀切禁止。

### 应用题（要点）
17. (1) GRANT SELECT ON TABLE Student TO U1 WITH GRANT OPTION;
   (2) GRANT UPDATE(Sname, Sage) ON TABLE Student TO U2;
   (3) REVOKE SELECT ON TABLE Student FROM U1 CASCADE;

18. (1) CREATE ROLE Teacher_Role;
   (2) GRANT SELECT, UPDATE ON TABLE SC TO Teacher_Role;
   (3) GRANT Teacher_Role TO T1, T2;

### 授权传播链分析题（要点）
19. (1) A、B、C、D、E 都拥有权限。
   (2) B 和 C 失去权限。D 通过 E 仍然拥有权限。A、E 保留。

20. 方案 A：3 条 REVOKE / 3 条 GRANT。方案 B：1 条 REVOKE / 1 条 GRANT。

### 辨析题
21. 错误。WITH GRANT OPTION 用于对象权限传播；WITH ADMIN OPTION 用于角色或系统权限传播。

22. 错误。CONNECT 只能登录，创建表需要 RESOURCE 或 DBA 权限。

23. 错误。审计是事后审查，不能阻止非法操作发生。

### 纠错题
24. (1) 对象权限应使用 WITH GRANT OPTION，非 WITH ADMIN OPTION。
   (2) INSERT 不能指定列，只有 UPDATE 和 REFERENCES 可以。
   (3) 角色名不应加引号。

### 案例分析题（要点）
25. (1) MAC。系统根据安全标记自动控制访问，且禁止用户传播权限。
   (2) 不能。DAC 允许用户传播权限，无法强制禁止。
   (3) "向下读"：军官可读机密及以下级别；"向上写"：用户不能将数据写入低于其级别的对象。
