# 第十章 数据库恢复技术 — 选择题解析

---

### 第 1 题

**答案：B**

**解析：**

事务 T 执行了 `A = A - 1000` 后断电，日志里有 BEGIN 但**没有 COMMIT**——说明 T **未提交**。

恢复原则：**未提交的事务必须 UNDO**（撤销），因为它的修改不该保留。

A 错：REDO 是给**已提交但可能没写入磁盘**的事务用的，T 没提交不该 REDO。
B 正确：T 未提交，A 的修改要撤销（UNDO），让 A 恢复原值。选 B。
C 错：不能"不操作"——A 可能已经被改到磁盘了，必须主动撤销。
D 错：UNDO 和 REDO 不会同时用在同一个事务上。

> 已提交 → REDO（重做确保写入）；未提交 → UNDO（撤销回到原状）。T 没 COMMIT，所以 UNDO。

### 第 2 题

**答案：B**

**解析：**

A 错：说反了。UNDO 针对**未提交**事务，REDO 针对**已提交**事务。
B 正确：UNDO 和 REDO 都依赖**日志文件**——UNDO 用日志里的旧值回滚，REDO 用日志里的新值重做。选 B。
C 错：UNDO 需要**旧值**（前像），REDO 需要**新值**（后像），不是 UNDO 更复杂。
D 错：REDO 需要**新值**，不是旧值。

> 日志里存两样：旧值（前像）给 UNDO 用，新值（后像）给 REDO 用。缺一不可。

### 第 3 题

**答案：B**

**解析：**

检查点技术：检查点时把内存缓冲区的日志和修改都写入磁盘，并记录当前活跃事务。

A 错：不是"只需扫描检查点之后的日志"——检查点时活跃的事务，要**回扫到它们的 BEGIN**，可能涉及检查点之前的日志。
B 正确：检查点之前**已提交**的事务，其修改在检查点时已写入磁盘，**不需要 REDO**。选 B。
C 错：检查点时活跃的事务**不一定**要 UNDO——如果它在 T2 前已提交，要 REDO；只有未提交的才 UNDO。
D 错：检查点**减少**日志扫描范围，不是完全避免。

> 检查点是一道"存档线"：存档前已提交的事务不用管了（已落盘），只处理存档时还活跃和存档后的事务。

### 第 4 题

**答案：D**

**解析：**

A 正确：静态转储期间不允许事务运行，数据一致性好。
B 正确：动态转储期间允许事务运行，不影响业务。
C 正确：静态转储时数据库静止，转储的就是一致状态，**不需要日志**配合即可恢复到转储点。
D **错误**：动态转储**需要日志文件**配合——因为转储时数据库还在变，转储出来的数据可能不一致，要用日志把转储期间的事务重做或撤销才能恢复一致状态。选 D。

> 静态转储 = 停机备份，数据干净不用日志；动态转储 = 边运行边备份，数据可能乱，必须靠日志"补全"。

### 第 5 题

**答案：B**

**解析：**

介质故障恢复：用最近一次**海量转储** + 之后的所有**增量转储** + 日志。

周三发生介质故障：
- 周日的海量转储是基础（完整备份）
- 周一、周二、周三的增量转储补上这三天的变化

B 正确：周日的海量转储 + 周一到周三的增量转储。选 B。
A 错：只有周日海量，漏了周一到周三的变化。
C 错：只有增量没基础，恢复不了。
D 错：漏了周一、周二的增量。

> 海量转储是"全量存档"，增量转储是"补丁"。恢复时：全量存档 + 之后的每个补丁，一个都不能少。

### 第 6 题

**答案：B**

**解析：**

**先写日志后写数据库**（Write-Ahead Logging, WAL）是恢复的核心原则。

A 错：先写日志不是为了性能，是为了**正确性/可恢复性**。
B 正确：先写日志保证即使写数据库时故障，也能靠日志恢复——日志里有完整记录，能 UNDO/REDO。选 B。
C 错：先写数据库后写日志，写日志时故障会导致**数据已改但日志没记**，无法恢复——这是必须避免的。
D 错：WAL 原则适用于**所有故障**（系统故障、介质故障），不只系统故障。

> 日志是"证据"，数据库是"现场"。先留证据再改现场——这样现场改到一半出事，靠证据能还原。

### 第 7 题

**答案：B**

**解析：**

ACID 四特性：

- **A 原子性**：事务要么全做要么全不做
- **C 一致性**：事务执行使数据库从一个一致性状态转到另一个一致性状态
- **I 隔离性**：并发事务互不干扰
- **D 持续性**：提交后永久保存

题目描述"从一个一致性状态转变到另一个一致性状态"对应**一致性**。

B 正确：选 B。

> 一致性 = 事务是"合法的状态转换"，前后都是合法状态。转账前后总金额不变就是一致性。

### 第 8 题

**答案：B**

**解析：**

A 错：数据库镜像不能**完全替代**转储，二者互补——镜像是实时复制，转储是定期备份。
B 正确：数据库镜像实时复制数据，主库故障时可**快速切换**到镜像库，保证可用性。选 B。
C 错：数据库镜像**需要日志**配合（同步镜像状态）。
D 错：镜像库**可以写入**（互为镜像时），不是只能查询。

> 镜像 = 主库的"实时替身"。主库倒了，替身立刻顶上。但替身不能替代"定期存档"（转储）。

### 第 9 题

**答案：A**

**解析：**

事务违反完整性约束被 DBMS 自动 ROLLBACK——这是**事务内部故障**。

事务内部故障：事务在运行中因**自身原因**（违反完整性、运算溢出、死锁被撤等）无法继续，由 DBMS 自动回滚。

A 正确：选 A。
B 错：系统故障是整个系统（如断电、OS 崩溃）导致的，不是单个事务的问题。
C 错：介质故障是磁盘损坏，和事务违反约束无关。
D 错：病毒是恶意破坏，不是事务自动回滚。

> 事务自己犯错（违约/溢出）被回滚 = 事务内部故障。整个系统崩了才是系统故障。

### 第 10 题

**答案：B**

**解析：**

日志文件两种格式：以记录为单位、以数据块为单位。

A 错：以记录为单位的日志不只记录事务标识和操作类型，还记录**旧值（前像）和新值（后像）**。
B 正确：以数据块为单位的日志记录**被更新的整个数据块**，不记录具体字段——它把整个块的旧值新值都存下来。选 B。
C 错：INSERT 操作的**旧值为空**（插入前没有数据），新值不为空。
D 错：DELETE 操作的**新值为空**（删除后没有数据），旧值不为空。说反了。

> INSERT：旧空新有；DELETE：旧有新空；UPDATE：旧有新有。记这个规律：操作前没有→旧空，操作后没有→新空。
