### 第 1 题

**答案：B**

**解析：**

这题考的是**可行性研究的三类划分**——技术/经济/操作（社会可行性是"操作可行性"的旧称，法律可行性常归入操作可行性或单列）。

教材标准的三类可行性：**技术可行性（Technical）+ 经济可行性（Economic）+ 操作可行性（Operational）**。技术 = 现有技术能否实现；经济 = 是否值得投资；操作 = 用户/组织能否使用。

- A 错：用"社会可行性"代替"操作可行性"——社会可行是旧称或操作可行的子集，**不标准**。
- B 正确：技术、经济、操作——教材标准答案。
- C 错：用"法律可行性"代替"经济可行性"——法律是**子维度**（如隐私法规），不是三大类之一。
- D 错：缺少技术可行性——任何项目**必须先技术可行**才能谈经济/法律。

> 三类可行性口诀：**"技经操"（技术+经济+操作）**。法律可行性通常作为"操作可行性"的子维度考虑。

### 第 2 题

**答案：C**

**解析：**

这题考的是**成本估算方法的适用场景**——COCOMO 模型是"代码行数已知"场景下的最佳选择。

4 个项目特征：①规模明确（10 万行）→ 适合参数化模型 ②无历史数据 → 排除类比估算 ③工期紧无详细需求 → 排除功能点（FP 需要详细功能列表） ④粗略但可信 → 需标准化模型。

**COCOMO 模型**（Constructive Cost Model）：以代码行数 + 模式系数（有机/嵌入/半独立）做参数化估算。**最适合规模已明确、有基本技术信息、需粗略但标准化的项目**。

- A 错：专家判断法适用于**信息极少**的场景（甚至规模都不明），且主观性大、不"可信"。
- B 错：类比估算需要**历史项目数据**——特征 ② 明确"无历史数据"，不适用。
- C 正确：COCOMO 用代码行数（已知 10 万行）做参数化估算，最匹配 4 个特征。
- D 错：FP 功能点需要**详细功能列表**——特征 ③ 明确"工期紧无详细需求分析"，不适用。

> 成本估算方法选用口诀：**"代码行→COCOMO；功能→FP；历史→类比；极少信息→专家"**。题干给出 4 个特征时，**找"匹配多数"的那个**。

### 第 3 题

**答案：A**

**解析：**

这题考的是**DFD 父图与子图的平衡原则**——子图作为整体必须保持与父图 P1 处理**完全相同的输入输出**。

DFD 平衡原则（★必考）：**父图中的某个处理 P1 的输入数据流集合 = 子图作为整体的输入数据流集合；输出同理**。子图内部可以怎么分解都行，但作为"黑盒"必须与父图等价。

题干：父图 P1 有 3 个输入（a, b, c）+ 2 个输出（x, y）。子图分解为 P1.1, P1.2, P1.3 三个子处理。子图作为整体，**输入必须有 a, b, c 三个，输出必须有 x, y 两个**——这就是平衡。

- A 正确：子图输入 = 父图 P1 输入，子图输出 = 父图 P1 输出（**完全等价**）。
- B 错：输出可调整——破坏平衡，父图与子图不一致。
- C 错：输入可调整——同样破坏平衡。
- D 错：输入输出都可调整——**完全破坏平衡**，子图与父图无法对应。

> DFD 平衡原则口诀：**"子=父"（子图作为黑盒与父图 P1 等价）**。常见错误：把"子图内部调整"和"子图整体调整"混淆——内部可以自由调整，整体必须守恒。

### 第 4 题

**答案：A**

**解析：**

这题考的是**数据字典 4 类元素**的精确定义。

数据字典（DD）4 类元素：
- **数据流（Data Flow）**：系统中**流动**的数据结构
- **数据存储（Data Store）**：**存放**的数据结构
- **数据处理（Process）**：处理逻辑（不在数据字典中描述逻辑，只描述数据）
- **数据项（Data Item）**：不可再分的数据**基本单元**

题干问"描述系统中流动的数据结构"——核心是"流动"——**数据流**。

- A 正确：数据流 = 系统中流动的数据结构。
- B 错：数据存储是"存放"的结构，不是"流动"的。
- C 错：数据处理是处理逻辑，不是数据结构。
- D 错：数据项是基本单元，不是"系统流动的"——数据项是数据的最小单位。

> 数据字典 4 类元素口诀：**"流存处项"（数据流+数据存储+数据处理+数据项）**，各自对应"动/静/处理/原子"。

### 第 5 题

**答案：B**

**解析：**

这题考的是**系统流程图 vs DFD 数据流图**的本质区别。

两种图的核心区别：
- **系统流程图**：用矩形框表示"操作/程序"，箭头表示"操作间先后顺序"——**关注控制流/操作顺序**
- **DFD 数据流图**：用圆/椭圆表示"加工"，箭头表示"数据流"——**关注数据如何流动和被加工**

题干：(I) 矩形框 + 操作先后顺序 → **系统流程图**；(II) 圆/椭圆 + 数据流 → **DFD 数据流图**。

- A 错：I 和 II 颠倒。
- B 正确：I 是系统流程图，II 是 DFD。
- C 错：两图本质不同（控制流 vs 数据流），不是粒度问题。
- D 错：两图本质不同。

> 系统流程图 vs DFD 口诀：**"矩形+控制流=系统流程图；圆+数据流=DFD"**。常见错误：把"操作"和"数据"搞混——系统流程图的箭头是"操作步骤"，DFD 的箭头是"数据"。

### 第 6 题

**答案：A**

**解析：**

这题考的是**经济可行性核心指标**——NPV/ROI/投资回收期的判别规则。

4 个核心指标：
- **NPV（净现值）**：NPV > 0 → **经济可行**（收益现值超过成本现值）
- **ROI（投资回报率）**：ROI 越大 → 越**经济可行**（每元投入回报越多）
- **投资回收期**：回收期越**短** → 越**经济可行**（越快回本）

A 正确：NPV > 0 → 经济可行（教材标准判别规则）。

- A 正确。
- B 错：投资回收期越**短**说明经济上**越可行**（快回本），不是"越不可行"。
- C 错：ROI 越大越**可行**（每元回报高），不是"越不可行"。
- D 错：投资回收期越**长**说明经济上**越不可行**（慢回本）。

> 经济可行性指标口诀：**"NPV>0 可行；ROI 大可行；回收期短可行"**。三个指标方向一致，**都是越大（NPV/ROI）或越小（回收期）越可行**。

### 第 7 题

**答案：A**

**解析：**

这题考的是**COCOMO 模型 3 层结构**——精度与工作量关系。

COCOMO 3 层：
- **基本 COCOMO**：按代码行数 + 3 种模式系数（有机/嵌入/半独立），精度最低
- **中等 COCOMO**：基本 + **15 个调节因子**（产品/平台/人员/项目），精度中等
- **详细 COCOMO**：中等 + **各阶段具体调节因子**，精度最高

精度与工作量关系：**调节因子越多 → 精度越高 → 数据收集工作量越大**。

- A 正确：精度从高到低：详细 > 中等 > 基本。
- B 错：基本 COCOMO 精度最低，不是最高。
- C 错：中等不是最高的——详细比中等多了"各阶段具体调节因子"，精度更高。
- D 错：3 层精度不同——这是核心区别。

> COCOMO 3 层口诀：**"基中详，精度增工作量增"**。**详细 COCOMO 工作量是基本的几倍**，但精度高。考试题常考"哪个更精确"。

### 第 8 题

**答案：C**

**解析：**

这题考的是**可行性研究的 8 个标准步骤**——顺序记忆。

8 个标准步骤的正确顺序：
1. **调查**当前系统
2. **重新定义问题**（基于调查结果）
3. **导出高层逻辑模型**
4. **导出和评价供选择的解法**
5. **推荐行动方针**
6. **草拟开发计划**
7. **编制可行性研究报告**
8. **提交上级审查**

口诀：**"调查→定义→模型→评价→推荐→草拟→报告→审查"**。

- A 错：把"重新定义问题"放到了"模型"之后——逻辑反了，**先定义问题再建模**。
- B 错：把"推荐"放到"评价"之前——评价完才能推荐。
- C 正确：调查→定义→模型→评价→推荐→草拟→报告→审查。
- D 错：把"推荐"放到"评价"之前——**逻辑反了**。

> 8 步口诀：**"调定模评推草报审"**（调查→定义→模型→评价→推荐→草拟→报告→审查）。常见错误：把"定义问题"和"建模"顺序颠倒——**先想清楚问题是什么，再建高层模型**。

### 第 9 题

**答案：B**

**解析：**

这题考的是**FP（功能点）估算的适用场景**——早期需求模糊时的粗略估算。

FP 功能点估算是基于**用户视角的功能度量**（输入/输出/查询/内部文件/外部接口），**与具体技术实现无关**。适用于**需求早期、信息不完整、需要粗略但可信的成本估算**的场景。

- A 错：FP 用于**早期粗略估算**，不是"需求明确的详细估算"——需求明确时可用 LOC（代码行）法更直接。
- B 正确：FP 的核心优势就是**需求模糊时给出可比较的规模度量**。
- C 错：FP 用于**预估**而不是"反推"——反推是后验度量。
- D 错：FP 用于**软件项目**，不用于纯硬件项目。

> FP vs LOC 选用口诀：**"需求模糊→FP；规模明确→LOC/COCOMO"**。FP 的最大价值是**早期 + 与技术无关**，让架构师/项目经理在需求阶段就能给老板"大致多少钱"。

### 第 10 题

**答案：A**

**解析：**

这题考的是**3 种流程图的精确辨析**——DFD / 系统流程图 / 程序流程图。

3 种图的核心区别：
- **DFD 数据流图**：圆/椭圆 = 加工；箭头 = **数据**；关注"数据怎么流动和被加工"
- **程序流程图**：方框 = 语句；菱形 = 判断；关注"代码执行顺序"
- **系统流程图**：矩形框 = 操作/程序；箭头 = **操作先后**；关注"系统各操作间顺序"

题干：(I) 圆/椭圆 + 数据流 → **DFD**；(II) 方框+菱形 + 执行顺序 → **程序流程图**；(III) 矩形框 + 操作先后 → **系统流程图**。

- A 正确：I=DFD、II=程序流程图、III=系统流程图。
- B 错：I 和 II 颠倒。
- C 错：II 和 III 颠倒。
- D 错：I 和 III 颠倒。

> 3 种流程图速记口诀：**"圆数据=DFD，菱形判断=程序流，矩形控制=系统流"**。考试题常混 3 种图，**核心是看"箭头代表什么"**（数据 vs 控制 vs 执行顺序）。
