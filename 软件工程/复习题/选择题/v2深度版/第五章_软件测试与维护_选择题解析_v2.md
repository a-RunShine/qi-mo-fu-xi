### 第 1 题

**答案：A**

**解析：**

这题考的是**6 种逻辑覆盖的关系**——从弱到强逐步严格。

6 种逻辑覆盖从弱到强：
- **语句覆盖**：每条语句至少执行一次（**最弱**）
- **判定覆盖**：每个判定（if/case）的真假分支至少各走一次
- **条件覆盖**：每个判定的每个条件的真假值至少各出现一次
- **判定条件覆盖**：判定 + 条件同时满足（简单叠加）
- **条件组合覆盖**：所有条件的**真值组合**至少各出现一次
- **路径覆盖**：所有可能的执行路径至少各走一次（**最强但常不可达**——循环/分支组合爆炸）

- A 正确：语句覆盖最弱，路径覆盖最强——但路径覆盖常因循环/分支组合**不可达**。
- B 错：条件覆盖不一定比判定覆盖严格——条件覆盖可能漏判定分支。
- C 错：判定条件覆盖**不是**简单叠加——是要求两者**同时**满足。
- D 错：路径覆盖是**所有路径**，不只是"包含语句覆盖的测试点"。

> 6 种覆盖口诀：**"语判条判条条路"**（语句→判定→条件→判定条件→条件组合→路径）。常见错误：把"判定条件覆盖"和"条件组合覆盖"搞混——前者是"判定+条件都覆盖"，后者是"所有条件组合都覆盖"。

### 第 2 题

**答案：B**

**解析：**

这题考的是**黑盒测试两种核心方法**——等价类划分 vs 边界值分析。

- **等价类划分**：将输入域划分为若干"等价类"，每类选**少数代表值**测试——适用于**所有输入**
- **边界值分析**：专门测试等价类**边界附近**的极值（如 0、1、150、151）

两者**互补不互斥**：等价类 + 边界值 = 完整黑盒测试。**边界值不可替代等价类**（边界是等价类的子集）。

- A 错：等价类划分选"代表值"测试，不是"所有输入"——是抽样不是穷举。
- B 正确：边界值分析专门测边界附近的特殊情况。
- C 错：两者**不是互斥**——是互补的，常配合使用。
- D 错：边界值不可替代等价类——边界值是"边界子集"测试。

> 等价类 vs 边界值口诀：**"等价类抽代表、边界值挑极值、两者结合最稳"**。常见错误：把边界值看作"比等价类更高级"——其实**两者层级不同**，边界值是"等价类的子集测试"。

### 第 3 题

**答案：A**

**解析：**

这题考的是**测试 4 层次**——单元→集成→系统→验收。

测试 4 层次标准顺序：
- **单元测试**：测试模块**内部**（单个函数/类）
- **集成测试**：测试**模块间接口**（组合模块测试接口）
- **系统测试**：测试**完整系统**（所有模块组合后的端到端流程）
- **验收测试**：用户**在实际环境**测试（UAT）

题干：
- I：模块 A 内部每个函数单独测试 → **单元**
- II：A 和 B 组合测试接口 → **集成**
- III：所有模块组合完整端到端流程 → **系统**
- IV：用户实际使用环境测试 → **验收**

- A 正确：单元→集成→系统→验收。
- B 错：颠倒——单元在集成之前。
- C 错：系统测试在集成测试之后（不是之前）——C 颠倒了系统测试与集成测试的顺序。
- D 错：集成和系统测试**不同**——前者测接口，后者测完整流程。

> 测试 4 层次口诀：**"元集系验"（单元→集成→系统→验收）**。每个层次测试目标不同：单元测代码正确性、集成测接口兼容性、系统测业务流程、验收测用户满意度。

### 第 4 题

**答案：A**

**解析：**

这题考的是**维护 4 类型**的精确判定。

软件维护 4 类型：
- **改正性维护**：修复**已发现的 bug**
- **适应性维护**：适应**外部环境变化**（如 OS 升级、硬件更换、法规变化）
- **完善性维护**：**增加新功能 / 改进性能**（用户新需求）
- **预防性维护**：**主动优化**代码以预防未来问题

题干：
- (a) 修复登录 bug → **改正性**
- (b) OS 升级需适配 → **适应性**
- (c) 增加 PDF 导出新功能 → **完善性**
- (d) 主动优化预防 bug → **预防性**

- A 正确：改正→适应→完善→预防。
- B 错：把适应性和改正性位置对调。
- C 错：把完善性和适应性的位置对调。
- D 错：把"主动优化（预防性）"和"适应性"对调。

> 4 类维护口诀：**"改适完预"（改正→适应→完善→预防）**。**"改"=修 bug、"适"=外部变化、"完"=加功能、"预"=主动优化**。常见错误：把"完善性"和"适应性"搞混——完善是**用户主动加功能**，适应是**外部环境倒逼**。

### 第 5 题

**答案：A**

**解析：**

这题考的是**α 测试 vs β 测试**的辨析。

- **α 测试**：在**开发方环境**（开发方公司内）测试，由**开发方人员/模拟用户**测试
- **β 测试**：在**用户真实环境**测试，由**真实用户**使用

题干：
- 阶段一：开发方公司内部测试，测试数据来自员工 → **α 测试**
- 阶段二：发布给外部真实用户在真实环境测试 → **β 测试**

- A 正确：阶段一 α，阶段二 β。
- B 错：颠倒。
- C 错：两阶段**测试主体和环境都不同**，不是同一个测试。
- D 错：两阶段**测试主体不同**——α 是开发方，β 是真实用户。

> α vs β 口诀：**"α 在家（开发方），β 在外（真实用户）"**。α 测试发现的问题**可以快速修复**（开发方控制环境）；β 测试发现的问题**收集后再修复**（用户环境无法实时调试）。

### 第 6 题

**答案：C**

**解析：**

这题考的是**回归测试的触发场景**——5 类工作场景都需要回归。

回归测试的核心：**任何代码/环境变更都可能引入新 bug**，因此每次变更后都要**重新跑相关测试**。

5 类工作场景的回归测试必要性：
- (a) 修 bug 后：必须跑——修 bug 可能引入新 bug
- (b) 加新功能后：必须跑——新功能可能影响旧功能
- (c) 重构代码后：必须跑——重构不改变行为但可能引入 bug
- (d) 升级第三方库后：必须跑——新库可能改变行为
- (e) 部署到生产前：必须跑——最后一道质量关

- A 错：(d)(e) 也需要回归。
- B 错：(a)(e) 也需要回归。
- C 正确：5 类全部都需要。
- D 错：除 (a) 外其他 4 类也需要。

> 回归测试触发口诀：**"变更后必回归，部署前必回归"**。只要有"代码/环境变化"，就要考虑回归测试。**漏掉回归是常见 bug 引入原因**。

### 第 7 题

**答案：C**

**解析：**

这题考的是**3 种调试策略**的精确定义。

3 种调试策略：
- **蛮力法**：通过**大量 print/日志/二分查找**找错误（最笨但通用）
- **回溯法**：从**错误位置向前回溯**到正确位置（适合小程序）
- **归纳法**：从**具体错误现象归纳**出一般规律（从特殊到一般）

题干：
- I：插入 print 打印变量值 → **蛮力法**
- II：从错误位置向前回溯 → **回溯法**
- III：从整体推测可能原因 → **归纳法**

- A 错：把"演绎"放进名字里——演绎是从一般到特殊，不是这 3 种。
- B 错：把"归纳"和"回溯"对调。
- C 正确：蛮力→回溯→归纳。
- D 错：完全打乱。

> 3 种调试策略口诀：**"蛮回归"（蛮力→回溯→归纳）**。**蛮力法**是"穷举日志"、**回溯法**是"倒着找"、**归纳法**是"从现象找规律"。

### 第 8 题

**答案：A**

**解析：**

这题考的是**等价类划分的本质**——按输入域划分有效/无效类。

等价类划分核心：把输入域划分为**有效等价类**（符合规格的输入）和**无效等价类**（不符合规格的输入），从每类中选**少量代表值**测试。

题干："输入年龄（1-150 之间的整数）"——有效等价类 = 1-150 整数，无效等价类 = 其他所有输入（负数、0、>150、非整数）。

- A 正确：有效等价类 = 1-150 整数；无效等价类 = 其他所有输入。
- B 错：把所有正整数都当有效——>150 是无效的（超出范围）。
- C 错：把任意整数当有效——负数是无效的。
- D 错：把"所有输入"当有效——显然错误（无效类存在）。

> 等价类划分口诀：**"有效 + 无效，按规格划分"**。**有效等价类**满足规格（输入域），**无效等价类**违反规格（输入域外）。常见错误：把"输入域的一部分"当有效——有效类**必须满足全部规格**。

### 第 9 题

**答案：A**

**解析：**

这题考的是**边界值分析的标准方法**——4 个值（上点 + 离点）。

边界值分析标准方法：测**上点**（边界值）和**离点**（紧邻边界的值）。
- **上点**：边界本身（如 1 和 150）
- **离点**：上点 ±1（如 0、1、150、151）

所以"1-150 整数"的有效边界值测试：**0、1、150、151** 共 **4 个值**。

- A 正确：0、1、150、151 共 4 个值（标准边界值分析方法）。
- B 错：1、150 只有 2 个——**只测上点没测离点**，不完整。
- C 错：0、1、149、150、151 多了 149——149 不是紧邻 150 的离点。
- D 错：1、2、149、150 测的是"内部点"——不是边界。

> 边界值分析口诀：**"上点 + 离点（±1），共 2N 个值（N = 边界数）"**。本题 1 和 150 是 2 个边界，2×2 = **4 个值**（0、1、150、151）。常见错误：把"内部点"（如 50、100）当边界值测试——**内部点属于等价类，不属于边界值**。

### 第 10 题

**答案：A**

**解析：**

这题考的是**单元测试 vs 集成测试**的本质区别——目标不同。

- **单元测试**：测试**单个模块内部**（函数/类），由**开发**执行，使用**白盒**技术
- **集成测试**：测试**模块间接口**（A 调用 B 是否正确），由**开发或测试**执行，使用**灰盒/黑盒**

两者都由**开发**执行（不是用户），都在**编码后**（不是编码前），都有**明确的测试目标**（不是无区别）。

- A 正确：单元测试关注模块内部，集成测试关注模块间。
- B 错：两者都**由开发执行**——用户执行的是验收测试。
- C 错：两者都在**编码后**——编码前是需求分析阶段。
- D 错：两者测试目标**完全不同**——内部 vs 接口。

> 单元 vs 集成测试口诀：**"单元测内部、集成测接口、系统测流程、验收测用户"**。注意**测试主体**是开发（单元/集成/系统）和用户（验收），**不是反过来**。
