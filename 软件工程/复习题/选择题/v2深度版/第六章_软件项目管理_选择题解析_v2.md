### 第 1 题

**答案：B**

**解析：**

这题考的是**CMM/CMMI 5 级成熟度模型**——已定义级的核心是"过程文档化标准化"。

5 级从低到高：
- **①初始级**：过程混乱，成功靠个人英雄——**不是**"已制度化"
- **②可重复级**：基本项目管理过程制度化，可重复成功
- **③已定义级**：过程**文档化、标准化**，所有项目用统一过程
- **④定量管理级**：过程数据**收集+分析**，用数据决策
- **⑤优化级**：持续改进过程（不是"自动化无需人为改进"——仍是人为驱动）

- A 错：初始级是"混乱靠个人"，不是"已制度化"。
- B 正确：已定义级 = 过程文档化标准化。
- C 错：定量管理级是"数据已收集+分析"，不是"未分析"。
- D 错：优化级是"持续改进"，仍是人为驱动，不是"自动化无需人为"。

> CMM 5 级口诀：**"初可已定优"（初始→可重复→已定义→定量管理→优化）**。**"已定义"=过程文档化**是分水岭——之前靠"人"管项目，之后靠"过程"管项目。

### 第 2 题

**答案：C**

**解析：**

这题考的是**关键路径 CPM 计算**——找最长路径即关键路径。

AOE 网络图所有路径：
- A→B→D→E = 3+4+2 = 9 天
- A→B→E = 3+5 = 8 天
- A→C→D→E = 2+6+2 = **10 天**（最长）

关键路径 = **A→C→D→E**，总工期 **10 天**。

- A 错：关键路径错（A→B→D→E 是 9 天不是关键路径），总工期错（12 天）。
- B 错：A→B→E 是 8 天，不是关键路径也不是 10 天。
- C 正确：关键路径 A→C→D→E = 10 天。
- D 错：路径对但总工期错（9 天不是 10 天）。

> 关键路径计算口诀：**"列出所有路径，求最长就是关键"**。关键路径上**任何任务延期都导致总工期延期**，非关键路径任务有"浮动时间"。

### 第 3 题

**答案：A**

**解析：**

这题考的是**风险管理 5 步骤**——识别→分析→应对→监控→沟通。

风险管理 5 步骤标准顺序：
1. **风险识别**：列出所有可能的风险
2. **风险分析**：评估每个风险的可能性 + 影响
3. **风险应对计划**：对每个风险制定应对策略
4. **风险监控**：跟踪已识别风险 + 发现新风险
5. **风险沟通**：在团队/干系人间沟通风险状态

- A 正确：识别→分析→应对→监控→沟通（标准顺序）。
- B 错：把"监控"放到"应对"之前——**错**，应先应对后监控。
- C 错：把"分析"放到"识别"之前——**错**，必须先识别才能分析。
- D 错：把"应对"放到"分析"之前——**错**，必须先分析才能应对。

> 风险管理 5 步口诀：**"识分应监沟"（识别→分析→应对→监控→沟通）**。**沟通贯穿始终**，但作为"独立步骤"是第 5 步。

### 第 4 题

**答案：A**

**解析：**

这题考的是**风险应对 4 策略**——缓解/规避/转移/开拓。

风险应对 4 策略：
- **缓解（Mitigate）**：**降低风险概率或影响**（如备份人员、测试、知识共享）
- **规避（Avoid）**：**改变计划消除风险**（如采用敏捷应对需求变更）
- **转移（Transfer）**：**将风险转给第三方**（如买保险、外包、采购商业组件）
- **开拓（Exploit）**：**主动追求**风险带来的机会（如多元化产品组合抓住新市场机会）

题干：
- (a) 关键人员离职——培养备份 + 知识共享 → **降低风险** → **缓解**
- (b) 需求变更——采用敏捷 + 模块化 → **改变方法** → **规避**
- (c) 技术难题——购买商业组件 → **转给第三方** → **转移**
- (d) 行业衰退——多元化产品组合 → **主动抓住机会** → **开拓**

- A 正确：缓解、规避、转移、开拓。
- B 错：把规避和开拓对调——多元化是**主动开拓机会**不是规避。
- C 错：把缓解和规避对调。
- D 错：完全打乱。

> 4 策略口诀：**"缓规转开"（缓解→规避→转移→开拓）**。**"缓解"=降低、"规避"=改变计划、"转移"=交给别人、"开拓"=主动追求机会**。

### 第 5 题

**答案：B**

**解析：**

这题考的是**PERT vs CPM** 的区别——一个应对不确定工期，一个应对确定工期。

- **CPM（关键路径法）**：任务工期**确定**（如"编码 5 天"），用于传统项目
- **PERT（计划评审技术）**：任务工期**不确定**（如"编码 5-10 天"），用**三点估算**（最乐观 + 最可能 + 最悲观）取期望值

两者**不是完全相同**（核心区别在工期是否确定），**PERT 不只算时间**（也考虑风险）。

- A 错：颠倒了——PERT 用概率工期，CPM 用确定工期。
- B 正确：PERT 应对时间不确定，CPM 应对时间确定。
- C 错：本质**不同**——工期处理方式根本不同。
- D 错：PERT 也**考虑风险**（三点估算的本质就是风险分析）。

> PERT vs CPM 口诀：**"PERT 三点估工期（不确），CPM 单点算路径（确定）"**。**考试题常考"哪个用三点估算"——答 PERT**。

### 第 6 题

**答案：B**

**解析：**

这题考的是**配置管理"基线"的作用**——变更的参考点。

**基线（Baseline）**是经过正式评审批准的配置项"快照"，作为后续**变更的参考点**。变更基线需要**走变更控制流程**（如 CCB 评审）。

- A 错：基线**不是"最终交付版本"**——基线可在任何阶段建立（需求/设计/编码/测试）。
- B 正确：基线是变更的参考点，变更前需评估对基线的影响。
- C 错：基线**不是"工作备份"**——备份是临时存储，基线是正式发布点。
- D 错：基线**不是"测试环境快照"**——基线是配置管理概念，与测试环境无关。

> 基线口诀：**"评审通过立基线，变更走流程"**。**基线建立 = 配置项正式发布**，**基线变更 = 走变更控制**。常见错误：把"基线"当"备份"——基线是**管理概念**，备份是**存储概念**。

### 第 7 题

**答案：B**

**解析：**

这题考的是**SQA（软件质量保证）的核心目标**——保证过程被遵循从而间接保证产品质量。

SQA 的核心目标：**保证软件开发过程被遵循**——通过**审计/评审过程**是否按标准过程执行。**间接**保证产品质量（过程对了，产品自然好）。

SQA **不是**：
- 不是"零缺陷"——任何过程都不可能零缺陷
- 不是"测试执行"——测试是 QC（质量控制）的事
- 不是"修 bug"——修 bug 是开发的事

- A 错："零缺陷"是**不现实目标**。
- B 正确：保证过程被遵循 → 间接保证产品质量。
- C 错：编写测试用例是**QC**（质量控制）的事。
- D 错：直接修 bug 是**开发**的事，SQA 是审计过程。

> SQA vs QC vs 测试口诀：**"SQA 管过程、QC 管产品、测试是 QC 的子集"**。SQA 关注"**做对的事**"，QC 关注"**做对的事做对了吗**"。

### 第 8 题

**答案：A**

**解析：**

这题考的是**进度压缩两种方法**——赶工 vs 快速跟进。

- **赶工（Crashing）**：**增加资源**（如加人、加设备）压缩关键路径任务工期（成本↑，可能引入协调问题）
- **快速跟进（Fast-Tracking）**：**并行原串行任务**（让本应串行的任务并行）（可能引入返工风险）

题干：
- 方案 P：增加开发人员并行工作 → **赶工**
- 方案 Q：调整任务顺序让串行并行 → **快速跟进**

- A 正确：方案 P 是赶工，方案 Q 是快速跟进。
- B 错：颠倒。
- C 错：两个方案本质不同。
- D 错：两个方案本质不同。

> 赶工 vs 快速跟进口诀：**"赶工加资源，快速并行串"**。**赶工=资源换时间（成本↑），快速跟进=并行换时间（风险↑）**。

### 第 9 题

**答案：A**

**解析：**

这题考的是**QA（质量保证）vs QC（质量控制）**的本质区别。

- **QA（质量保证）**：关注**过程**（process）——通过保证过程正确来预防缺陷
- **QC（质量控制）**：关注**产品**（product）——通过测试/检查发现产品中的缺陷

两者**都关注"质量"**，但路径不同：QA 是**预防性**（事前），QC 是**检测性**（事中/事后）。

- A 正确：QA 关注过程，QC 关注产品。
- B 错：颠倒。
- C 错：QA 不关注产品。
- D 错：QC 不只关注过程——QC 也关注产品。

> QA vs QC 口诀：**"QA 防患于未然（过程），QC 验身于既成（产品）"**。**SQA 团队 = QA，测试团队 = QC**。

### 第 10 题

**答案：C**

**解析：**

这题考的是**风险类型分类**——已知风险 vs 未知风险。

风险分类（PMI 经典）：
- **已知风险（Known risk）**：**已识别并可分析**的风险——可提前识别 + 可分析可能性 + 可制定应对（如"未掌握新技术""需求不明确""供应商可能破产""士气可能下降"——这些都可识别、可分析、可应对）
- **未知风险（Unknown risk）**：**无法提前识别**的风险（如"明年可能地震"——极小概率不可预测）

题干 5 类风险：
- (a) 未掌握新技术 → **可识别**（团队技能评估能发现）→ 已知
- (b) 需求不明确 → **可识别** → 已知
- (c) 核心供应商可能破产 → **可识别**（供应商有经营状况可查询）→ 已知
- (d) 团队士气可能下降 → **可识别**（士气可观察）→ 已知
- (e) 明年地震 → **不可识别**（极小概率不可预测）→ **未知**

- A 错：实际有 4 个已知风险（不是 2 个）——只算 (a)(b) 漏了 (c)(d)。
- B 错：实际有 4 个已知风险（不是 3 个）——只算内部风险漏了 (c) 供应商（供应商虽在外部但风险可识别）。
- C 正确：4 个含供应商在内的风险都可识别为已知风险。
- D 错：地震是未知风险（极小概率不可预测），不是已知。

> 已知 vs 未知风险口诀：**"能识别的是已知，无法预测的是未知"**。**已知风险 = 可应对（写计划），未知风险 = 买保险/留 buffer**。注意**核心供应商破产虽不可控，但可识别**——"可识别"和"可控制"是两回事，不能因为"不可控"就归为未知。
